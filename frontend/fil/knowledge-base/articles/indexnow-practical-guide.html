IndexNow: Praktikal na Implementasyon at Mga Tip

15 Setyembre 2025 • 5 minutong pagbabasa
Bakit basahin: kung paano mabilis na maihahatid ang mga update sa mga search engine, kung kailan gagamitin ang changed-only, paano naiiba ang IndexNow sa sitemap ping, at kung paano maiiwasan ang mga karaniwang pagkakamali.

1) Ano ang IndexNow

Ang IndexNow ay isang protocol na nagbibigay-daan sa iyo na agad na ipaalam sa mga search engine ang tungkol sa mga pagbabago sa URL (paglikha, pag-update, pagtanggal). Suportado ng Bing, Yandex, Seznam at iba pang mga serbisyo. Ang kalamangan ay minimal na pagkaantala sa pag-index at mas kaunting ingay sa pag-crawl.

2) Pag-setup ng susi at pangunahing ping

  1. Gumawa ng isang susi (GUID) at ilagay ang file na YOUR_KEY.txt sa ugat ng site (/.well-known/ o ugat ng domain) gamit ang string ng susi.
  2. I-ping ang endpoint gamit ang URL at susi. Halimbawa (isang URL):
POST https://api.indexnow.org/indexnow Content-Type: application/json { "host": "www.serpmonn.ru", "key": "YOUR_KEY", "urlList": [ "https://www.serpmonn.ru/frontend/knowledge-base/articles/utm-complete-guide.html" ] }
Para sa mga batch, tukuyin hanggang 10,000 URL sa isang kahilingan o gumamit ng file feed na may URL file ng log ng pagbabago.

3) Changed-only mode

Ang ideya ay i-ping lamang ang mga file na talagang nagbago sa huling commit/deployment. Binabawasan nito ang ingay at paggamit ng quota.

  • Kunin ang listahan ng mga nagbago na landas mula sa VCS (git diff) o mga artifact ng build.
  • I-filter ayon sa mga pampublikong ruta (hindi kasama ang node_modules, .git, mga pribadong folder).
  • Pangkatin ayon sa mga domain/wika upang maiwasan ang pagpapadala ng hindi kaugnay na mga URL.
Sa produksyon, kapaki-pakinabang na magkaroon ng fallback: kung ang changed-only ay walang laman (halimbawa, hotfix para sa mga configuration), i-ping ang mga mahalagang sitemap.

4) IndexNow vs sitemap ping

  • Bilis: Mas mabilis na naghahatid ng mga update ang IndexNow, hindi naghihintay para sa cycle ng crawl; ang sitemap ping ay nagpapahiwatig ng pagkakaroon ng sitemap ngunit hindi mga pagbabago sa partikular na URL.
  • Ingay: IndexNow - nakatuon sa mga URL na nagbago; sitemap - pag-scan ng buong mapa (mas mahal para sa badyet ng crawl).
  • Pagiging maaasahan: Ang mga sitemap ay nananatiling base; Ang IndexNow ay nagdaragdag at nagpapabilis. Mas mainam na gamitin ang pareho.

5) Karaniwang mga pagkakamali

  • Ang susi ay hindi maa-access sa pamamagitan ng pampublikong URL o hindi tumutugma sa parameter na key.
  • Pag-ping sa mga pribado/404 na pahina, mga parameter nang walang canonicalization.
  • Masyadong madalas na pag-uulit ng parehong mga URL (throttle ang mga pag-uulit).
  • Hindi papansin ang multilinggwalismo (kailangang i-ping ang kaukulang na-localize na bersyon).

6) Mga FAQ at checklist

Ilang URL ang ipadala nang sabay-sabay?

Hanggang 10,000 sa isang batch - pinakamainam na pangkatin ayon sa uri/wika.

Dapat bang i-ping ang mga pagtanggal?

Oo - ipasa ang mga URL ng mga tinanggal na pahina upang mapabilis ang deindexing.

Paano kumpirmahin na ito ay gumagana?

Mga log ng tugon ng API (200/202), mga spike ng crawl sa mga log ng server, mas mabilis na paglitaw ng mga bagong URL.

  • Ang susi ay inilagay at naa-access sa pamamagitan ng HTTPS.
  • Kolektahin ang mga landas na changed-only mula sa CI/CD.
  • Pagdeduplicate at pag-throttle ng mga pag-uulit.
  • Fallback sa sitemap ping sa walang laman na batch.
: Makabuluhang pinapabilis ng IndexNow ang pag-index ng mga bago at binagong pahina. Pinagsama sa mga tradisyonal na sitemap, ito ay isang makapangyarihang tool para sa SEO optimization.
