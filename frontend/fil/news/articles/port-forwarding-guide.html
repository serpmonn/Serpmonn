Port Forwarding: Kumpletong Gabay 2025

Oktubre 29, 2025 ‚Ä¢ 15 minutong pagbabasa

Ang port forwarding ay isa sa mga pinaka-hinahanap at sabay na pinakamahirap na paksa para sa mga nagsisimula sa pagsasaayos ng home network. Sa gabay na ito, tatalakayin namin nang detalyado kung paano tamang i-set up ang port forwarding para sa anumang layunin: mula sa mga game server hanggang sa mga surveillance system.

üîç Ano ang Port Forwarding?

Port Forwarding ay isang teknolohiya na nagpapahintulot sa pag-redirect ng mga network request mula sa external port ng router patungo sa isang tiyak na port ng device sa iyong lokal na network.

Internet

Ang mga panlabas na kahilingan ay dumarating sa iyong router

Router

Nagre-redirect ng mga kahilingan sa kinakailangang device

Iyong Device

Tumatanggap at nagpoproseso ng mga kahilingan

‚ö†Ô∏è Mahalagang maunawaan

Ang port forwarding ay nagbubukas ng access sa iyong device mula sa internet. Laging sundin ang mga hakbang sa seguridad at huwag magbukas ng mga port na hindi kailangan!

üîß Paghahanda para sa Setup

  1. Tukuyin ang static IP address ng device

    Ang iyong device (PC, server, camera) ay nangangailangan ng permanenteng lokal na IP address upang malaman ng router kung saan i-forward ang traffic.

    Paano malaman ang IP address sa Windows:

    ipconfig

    Hanapin ang linya na "IPv4 Address" para sa iyong network connection

    Paano malaman ang IP address sa Linux/macOS:

    ifconfig

    O gamitin ang: ip addr show

  2. I-configure ang static IP

    Upang maiwasan ang pagbabago ng IP address, i-configure ito bilang static:

    • Sa router: Magtalaga ng static IP sa pamamagitan ng DHCP reservation
    • Sa device: Manu-manong i-configure ang static IP sa network settings

  3. Alamin ang external IP address

    Ito ang address na nakikita ng ibang mga user sa internet:

    Mga paraan upang malaman ang external IP:

    # Sa pamamagitan ng command line
curl ifconfig.me

# O bisitahin ang mga site:
# - whatismyipaddress.com
# - 2ip.ru
# - myip.com

  4. Tukuyin ang mga kinakailangang port

    Alamin kung aling mga port ang ginagamit ng iyong application o serbisyo. Ang impormasyong ito ay karaniwang nasa documentation.

üöÄ Step-by-step na tagubilin sa setup

  1. Mag-log in sa control panel ng router

    Buksan ang browser at ipasok ang IP address ng router. Karaniwan itong:

    • 192.168.1.1 (TP-Link, D-Link)
    • 192.168.0.1 (Netgear, ilang TP-Link)
    • 192.168.31.1 (Xiaomi)
    • 192.168.10.1 (Huawei)

    Ang username/password ay karaniwang nakalista sa sticker sa router.

  2. Hanapin ang section ng port forwarding

    Ang section ay maaaring may iba't ibang pangalan:

    • Port Forwarding
    • Virtual Servers
    • Port Forwarding
    • NAT
    • Firewall

  3. Punan ang mga parameter ng port forwarding

    Kailangan mong tukuyin ang:

    • Pangalan ng Serbisyo: Arbitraryong pangalan (halimbawa "Minecraft Server")
    • External Port: Port na bubuksan sa router
    • Internal Port: Port sa iyong device
    • Internal IP: Static IP ng iyong device
    • Protocol: TCP, UDP o Parehong

  4. I-save ang mga setting

    Matapos punan ang lahat ng field, pindutin ang "Save" o "Apply". Maaaring mag-restart ang router.

üì° Mga tagubilin para sa sikat na router

üîÑ TP-Link

  1. Mag-log in sa control panel (192.168.1.1)
  2. Pumunta sa: Advanced ‚Üí NAT Forwarding ‚Üí Virtual Servers
  3. Pindutin ang "Add New"
  4. Punan ang:
    • Service Type: Piliin mula sa listahan o ipasok ang iyong sarili
    • External Port: Port para sa access mula sa labas
    • Internal Port: Port sa device
    • IP Address: Lokal na IP ng device
    • Protocol: TCP/UDP/BOTH
  5. I-save at i-restart ang router

üõ°Ô∏è ASUS

  1. Mag-log in sa panel (192.168.1.1)
  2. Pumunta sa: Advanced Settings ‚Üí WAN ‚Üí Virtual Server/Port Forwarding
  3. I-enable ang "Enable Port Forwarding"
  4. Punan ang mga field:
    • Service Name: Pangalan ng serbisyo
    • Port Range: Saklaw ng port (halimbawa 25565-25565)
    • Local IP: IP ng iyong device
    • Local Port: Port ng device
    • Protocol: TCP/UDP/BOTH
  5. Pindutin ang "Add" at "Apply"

üîó D-Link

  1. Mag-log in sa panel (192.168.0.1)
  2. Pumunta sa: Advanced ‚Üí Port Forwarding
  3. Pindutin ang "Add"
  4. Punan ang:
    • Rule Name: Pangalan ng patakaran
    • External Port: External port
    • Internal Port: Internal port
    • Internal IP: IP ng device
    • Schedule: Always
    • Inbound Filter: Allow All
  5. I-save ang mga setting

üì∂ Zyxel Keenetic

  1. Mag-log in sa panel (my.keenetic.net o 192.168.1.1)
  2. Pumunta sa: Seguridad ‚Üí Port Forwarding
  3. Pindutin ang "Magdagdag ng panuntunan"
  4. Punan ang:
    • Paglalarawan: Pangalan ng serbisyo
    • Interface: Internet
    • Protocol: TCP/UDP
    • External port: Port para sa access mula sa labas
    • IP address ng tatanggap: Lokal na IP ng device
    • Internal port: Port sa device
  5. I-save ang mga pagbabago

‚úÖ Pag-verify ng port forwarding

  1. Gumamit ng mga online service

    Mga site para sa pag-check ng mga bukas na port:

    • portchecker.co
    • canyouseeme.org
    • 2ip.ru/port-scan/

  2. Pag-verify sa pamamagitan ng command line

    Pag-check mula sa ibang computer:

    # Gamitin ang telnet o nc (netcat) 
telnet iyong-external-ip port 
# O gamit ang nmap 
nmap -p port iyong-external-ip

  3. Pagsusuri ng serbisyo

    Subukang kumonekta sa iyong serbisyo mula sa ibang device sa labas ng lokal na network (halimbawa sa pamamagitan ng mobile internet).

‚úÖ Mga palatandaan ng matagumpay na setup

  • Ang port ay ipinapakita bilang "bukas" sa online checker
  • Maaari kang kumonekta sa serbisyo mula sa labas
  • Ang serbisyo ay tumatakbo nang matatag nang walang mga pagputol

üîß Paglutas ng mga karaniwang problema

üö´ Sarado pa rin ang port

  • Suriin ang firewall: Magdagdag ng exception sa Windows Firewall o ibang firewall
  • I-restart ang router: Minsan ang mga setting ay inilalapat lamang pagkatapos ng restart
  • Suriin ang IP address: Siguraduhin na ang device ay may tamang static IP
  • I-disable ang UPnP: Minsan ang UPnP ay sumasalungat sa manual na port settings

‚ö†Ô∏è Dynamic IP ng provider

Kung mayroon kang dynamic IP, maaari itong magbago. Mga solusyon:

  • Gumamit ng DDNS: Mga serbisyo tulad ng No-IP, DuckDNS
  • Suriin ang IP: Regular na i-update ang impormasyon tungkol sa kasalukuyang IP
  • Static IP: Mag-order ng static IP mula sa provider (bayad)

üîí Double NAT

Ang problema ay nangyayari kapag mayroon kang maraming router sa network:

  • Bridge mode: I-set up ang isa sa mga router sa bridge mode
  • DMZ: I-set up ang DMZ sa pangunahing router para sa pangalawang router
  • Forwarding sa pareho: I-set up ang forwarding sa parehong router

üõ°Ô∏è Mga hakbang sa seguridad

üîê Mga kritikal na patakaran

  1. Buksan lamang ang mga kinakailangang port - bawat bukas na port ay nagpapataas ng attack surface
  2. Gumamit ng kumplikadong mga password sa lahat ng serbisyo na naa-access mula sa labas
  3. Regular na i-update ang software - alisin ang mga vulnerabilities
  4. Subaybayan ang mga koneksyon gamit ang mga log ng router
  5. Isaalang-alang ang VPN bilang mas ligtas na alternatibo sa port forwarding

üéØ Mga pinakamahusay na kasanayan

  • Gumamit ng hindi karaniwang mga port upang mabawasan ang pansin ng mga scanner
  • I-set up ang awtomatikong pag-disable ng mga port ayon sa iskedyul
  • Regular na suriin ang mga log para sa kahina-hinalang aktibidad
  • Gumamit ng fail2ban o katulad para sa pag-block ng mga kahina-hinalang IP
