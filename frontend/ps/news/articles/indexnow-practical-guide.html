IndexNow: عملي پلي کړنه او لارښوونې

۱۵ سپتمبر ۲۰۲۵ • ۵ دقیقې لوستل
ولې ولولئ: څنګه چټک تازه معلومات پلټنې ماشینونو ته وړاندې کړئ، کله یوازې-تغیرشوي وکاروئ، IndexNow څنګه د sitemap ping څخه توپیر لري او څنګه معمولې تېروتنې څخه ځان وساتئ
مخونه
  1. IndexNow څه دی
  2. کليټ ترتیب او بنسټیز ping
  3. یوازې-تغیرشوي حالت: کله او څنګه
  4. IndexNow په مقابل کې sitemap ping
  5. معمولې تېروتنې
  6. په زړه پښو پوښتنې او چک لیست

۱) IndexNow څه دی

IndexNow یو پروتوکول دی چې تاسو ته اجازه ورکوي چې په فوري ډول پلټنې ماشینونه د URL بدلونونو (جوړول، تازه کول، ړنګول) په اړه خبر کړئ. د Bing، Yandex، Seznam او نورو خدماتو لخوا مالتړ شوی. ګټه لږترلږه انډکس وړونکي وخت او د کرول شور کمول دي.

۲) کليټ ترتیب او بنسټیز ping

  1. یو کليټ (GUID) جوړ کړئ او YOUR_KEY.txt فایل د ویب پاڼې په ریښه کې (/.well-known/ یا د ډومین ریښه) د کليټ سټرینګ سره ځای پرځای کړئ.
  2. د URL او کليټ سره پایټ ته ping واستوئ. مثال (یو واحد URL):
POST https://api.indexnow.org/indexnow Content-Type: application/json { "host": "www.serpmonn.ru", "key": "YOUR_KEY", "urlList": [ "https://www.serpmonn.ru/frontend/news/articles/utm-complete-guide.html" ] }
د ګروپونو لپاره، په یو غوښتنه کې تر ۱۰،۰۰۰ پورې URL وټاکئ یا د بدلون د لګ فایل URL سره د فایل فیډ کاروئ.

۳) یوازې-تغیرشوي حالت

فکر دا دی چې یوازې هغه فایلونو ته ping واستوئ چې په وروستي commit/استقرار کې واقعیا بدل شوي دي. دا شور او د کوټې کارول کموي.

  • د VCS (git diff) یا د جوړولو آرتیفیکټونو څخه د بدل شوي لارو لیست ترلاسه کړئ.
  • د عامو لارو له مخې فلټر کړئ (node_modules، .git، خصوصي فولډرونه له پامه وغورځوئ).
  • د غیر اړوند URL لیږلو څخه د مخنیوي لپاره د ډومین/ژبو له مخې ډله کړئ.
په تولید کې، د ریزرو درلودل ګټور دی: که یوازې-تغیرشوي تش وي (لکه د تنظیمونو لپاره تود ترمیم)، مهم سایټ مېپونو ته ping واستوئ.

۴) IndexNow په مقابل کې sitemap ping

  • چټکتیا: IndexNow تازه معلومات ګړندي وړاندې کوي، د کرول څرخ ته انتظار نه کوي؛ sitemap ping د سایټ مېپ شتون څرګندوي مګر د ځانګړي URL بدلونونه نه.
  • شور: IndexNow - د بدل شوي URLونو په لور موخه؛ سایټ مېپ - د ټولې نقشې اسکن (د کرول بودیجې لپاره ګران).
  • اعتبار: سایټ مېپونه اساس پاتې کیږي؛ IndexNow تکمیلوي او ګړندي کوي. د دواړو کارول غوره دي.

۵) معمولې تېروتنې

  • کليټ د عام URL له لارې لاسرسی وړ نه دی یا د key پیرامیټر سره سمون نه خوري.
  • د خصوصي/۴۰۴ پاڼو ping کول، د کانونیکل کولو پرته پیرامیټرونه.
  • د ورته URLونو ډیر ډیر تکرارونه (تکرارونه محدود کړئ).
  • د څو ژبو نادیده کول (اړین دی چې اړونده محلي شوي نسخه ته ping واستوئ).

۶) په زړه پښو پوښتنې او چک لیست

په یو وخت کې څو URL لیږم؟

په یو ګروپ کې تر ۱۰،۰۰۰ پورې - د ډول/ژبې له مخې په غوره ډول ډله کړئ.

ایا ړنګونې ping کړم؟

هو - د ړنګ شویو پاڼو URLونه د deindexing ګړندي کولو لپاره لیږدوئ.

څنګه تایید کړم چې کار کوي؟

API ځواب ریکارډونه (۲۰۰/۲۰۲)، د سرور ریکارډونو کې د کرول څوکۍ، نوي URLونو ګړندی څرګندیدل.

  • کليټ ځای پرځای شوی او د HTTPS له لارې لاسرسي وړ دی.
  • د CI/CD څخه یوازې-تغیرشوي لارې راټول کړئ.
  • د تکرارونو لرې کول او محدودول.
  • تش ګروپ کې sitemap ping ته ریزرو.
