Port Forwarding: Ghid Complet 2025

29 octombrie 2025 • 15 minute citire

Port forwarding este unul dintre cele mai solicitate și în același timp mai dificile subiecte pentru începătorii în configurarea rețelei de acasă. În acest ghid, vom discuta în detaliu cum să configurați corect port forwarding pentru orice scop: de la servere de jocuri până la sisteme de monitorizare.

Ce este Port Forwarding?

Port Forwarding este o tehnologie care permite redirecționarea cererilor de rețea de la portul extern al routerului către un port specific al unui dispozitiv din rețeaua dvs. locală.

Important de înțeles

Port forwarding deschide accesul la dispozitivul dvs. de pe internet. Respectați întotdeauna măsurile de securitate și nu deschideți porturi inutile!

Pregătire pentru configurare

  1. Determinați adresa IP statică a dispozitivului

    Dispozitivul dvs. (PC, server, cameră) are nevoie de o adresă IP locală permanentă, astfel încât routerul să știe unde să redirecționeze traficul.

    Cum să aflați adresa IP în Windows:

    ipconfig

    Căutați linia "Adresa IPv4" pentru conexiunea de rețea

    Cum să aflați adresa IP în Linux/macOS:

    ifconfig

    Sau folosiți: ip addr show

  2. Configurați IP static

    Pentru a evita modificări ale adresei IP, configurați-o static:

    • În router: Setați IP static prin rezervare DHCP
    • Pe dispozitiv: Configurați IP static manual în setările de rețea

  3. Aflați adresa IP externă

    Aceasta este adresa pe care o văd alți utilizatori pe internet:

    Modalități de a afla IP extern:

    # Prin linia de comandă
curl ifconfig.me

# Sau vizitați site-uri:
# - whatismyipaddress.com
# - 2ip.ru
# - myip.com

  4. Determinați porturile necesare

    Aflați ce porturi folosește aplicația sau serviciul dvs. Aceste informații se găsesc de obicei în documentație.

Instrucțiuni de configurare pas cu pas

  1. Autentificați-vă în panoul de control router

    Deschideți browserul și introduceți adresa IP a routerului. De obicei este:

    • 192.168.1.1 (TP-Link, D-Link)
    • 192.168.0.1 (Netgear, unele TP-Link)
    • 192.168.31.1 (Xiaomi)
    • 192.168.10.1 (Huawei)

    Numele de utilizator/parola implicită sunt de obicei listate pe eticheta routerului.

  2. Găsiți secțiunea port forwarding

    Secțiunea poate avea denumiri diferite:

    • Port Forwarding
    • Virtual Servers
    • Redirecționare Porturi
    • NAT
    • Firewall

  3. Completați parametrii port forwarding

    Trebuie să specificați:

    • Nume Serviciu: Nume arbitrar (ex: "Server Minecraft")
    • Port Extern: Portul care va fi deschis pe router
    • Port Intern: Portul pe dispozitivul dvs.
    • IP Intern: IP static al dispozitivului dvs.
    • Protocol: TCP, UDP sau Ambele

  4. Salvați setările

    După completarea tuturor câmpurilor, apăsați "Salvare" sau "Aplicare". Routerul poate reporni.

Instrucțiuni pentru routere populare

TP-Link

  1. Autentificați-vă în panoul de control (192.168.1.1)
  2. Accesați: Advanced → NAT Forwarding → Virtual Servers
  3. Faceți clic pe "Add New"
  4. Completați:
    • Service Type: Alegeți din listă sau introduceți personalizat
    • External Port: Port pentru acces extern
    • Internal Port: Port pe dispozitiv
    • IP Address: IP local al dispozitivului
    • Protocol: TCP/UDP/BOTH
  5. Salvați și reporniți routerul

ASUS

  1. Autentificați-vă în panou (192.168.1.1)
  2. Accesați: Advanced Settings → WAN → Virtual Server/Port Forwarding
  3. Activați "Enable Port Forwarding"
  4. Completați câmpurile:
    • Service Name: Nume serviciu
    • Port Range: Interval porturi (ex: 25565-25565)
    • Local IP: IP dispozitivului dvs.
    • Local Port: Port dispozitivului dvs.
    • Protocol: TCP/UDP/BOTH
  5. Faceți clic pe "Add" apoi "Apply"

D-Link

  1. Autentificați-vă în panou (192.168.0.1)
  2. Accesați: Advanced → Port Forwarding
  3. Faceți clic pe "Add"
  4. Completați:
    • Rule Name: Nume regulă
    • External Port: Port extern
    • Internal Port: Port intern
    • Internal IP: IP dispozitivului
    • Schedule: Always
    • Inbound Filter: Allow All
  5. Salvați setările

Zyxel Keenetic

  1. Autentificați-vă în panou (my.keenetic.net sau 192.168.1.1)
  2. Accesați: Securitate → Redirecționare Porturi
  3. Faceți clic pe "Adăugare regulă"
  4. Completați:
    • Descriere: Nume serviciu
    • Interfață: Internet
    • Protocol: TCP/UDP
    • Port extern: Port pentru acces extern
    • IP destinație: IP local al dispozitivului
    • Port intern: Port pe dispozitiv
  5. Salvați modificările

Verificarea funcționării port forwarding

  1. Utilizați servicii online

    Site-uri pentru verificarea porturilor deschise:

    • portchecker.co
    • canyouseeme.org

  2. Verificare prin linia de comandă

    Verificați de pe alt computer:

    # Folosiți telnet sau nc (netcat) 
telnet IP-EXTERN-DVS port 
# Sau folosind nmap 
nmap -p port IP-EXTERN-DVS

  3. Testarea serviciului

    Încercați să vă conectați la serviciul dvs. de pe alt dispozitiv în afara rețelei locale (ex: prin internet mobil).

Semne de configurare reușită

  • Portul apare ca "deschis" în verificatoare online
  • Vă puteți conecta la serviciu din exterior
  • Serviciul funcționează stabil fără întreruperi

Depanarea problemelor comune

Portul încă este închis

  • Verificați firewall-ul: Adăugați excepție în Windows Firewall sau alt firewall
  • Reporniți routerul: Uneori setările se aplică doar după repornire
  • Verificați adresa IP: Asigurați-vă că dispozitivul are IP static corect
  • Dezactivați UPnP: Uneori UPnP intră în conflict cu setările manuale de porturi

IP dinamic de la furnizor

Dacă aveți IP dinamic, acesta se poate schimba. Soluții:

  • Folosiți DDNS: Servicii precum No-IP, DuckDNS
  • Verificați IP-ul: Actualizați informații despre IP-ul curent regulat
  • IP static: Comandați IP static de la furnizor (plătit)

NAT dublu

Problema apare când aveți mai multe routere în rețea:

  • Mod bridge: Setați unul dintre routere în mod bridge
  • DMZ: Configurați DMZ pe routerul principal pentru al doilea router
  • Redirecționare pe ambele: Configurați redirecționarea pe ambele routere

Măsuri de securitate

Reguli critice

  1. Deschideți doar porturile necesare - fiecare port deschis mărește suprafața de atac
  2. Folosiți parole complexe pe toate serviciile accesibile din exterior
  3. Actualizați software-ul regulat - eliminați vulnerabilități
  4. Monitorizați conexiunile folosind jurnalele routerului
  5. Luați în considerare VPN ca alternativă mai sigură la port forwarding

Cele mai bune practici

  • Folosiți porturi non-standard pentru a reduce atenția scanerelor
  • Configurați dezactivarea automată a porturilor conform programului
  • Verificați jurnalele regulat pentru activitate suspectă
  • Folosiți fail2ban sau similar pentru a bloca IP-uri suspecte
