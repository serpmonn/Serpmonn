IndexNow: ব্যবহারিক বাস্তবায়ন এবং টিপস

১৫ সেপ্টেম্বর ২০২৫ • ৫ মিনিট পড়া
কেন পড়বেন: কিভাবে দ্রুত সার্চ ইঞ্জিনে আপডেট দিতে হয়, কখন শুধুমাত্র-পরিবর্তিত ব্যবহার করবেন, IndexNow কিভাবে সাইটম্যাপ পিং থেকে আলাদা এবং কিভাবে সাধারণ ভুলগুলি এড়াবেন।

1) IndexNow কি

IndexNow একটি প্রোটোকল যা আপনাকে URL পরিবর্তন (তৈরি, আপডেট, মুছে ফেলা) সম্পর্কে তাত্ক্ষণিকভাবে সার্চ ইঞ্জিনগুলিকে জানাতে দেয়। বিং, ইয়ানডেক্স, Seznam এবং অন্যান্য পরিষেবা দ্বারা সমর্থিত। সুবিধা হল ন্যূনতম ইন্ডেক্সিং বিলম্ব এবং কম ক্রল শোরগোল।

2) কী সেটআপ এবং বেসিক পিং

  1. একটি কী (GUID) তৈরি করুন এবং YOUR_KEY.txt ফাইলটি সাইটের রুটে (/.well-known/ বা ডোমেন রুট) কী স্ট্রিং সহ রাখুন।
  2. URL এবং কী দিয়ে এন্ডপয়েন্টে পিং করুন। উদাহরণ (একক URL):
POST https://api.indexnow.org/indexnow Content-Type: application/json { "host": "www.serpmonn.ru", "key": "YOUR_KEY", "urlList": [ "https://www.serpmonn.ru/frontend/knowledge-base/articles/utm-complete-guide.html" ] }
ব্যাচের জন্য, একটি অনুরোধে 10,000 পর্যন্ত URL উল্লেখ করুন বা একটি পরিবর্তন লগ ফাইল URL সহ একটি ফাইল ফিড ব্যবহার করুন।

3) শুধুমাত্র-পরিবর্তিত মোড

ধারণা হল শুধুমাত্র সেই ফাইলগুলিকে পিং করা যা শেষ কমিট/ডেপ্লয়মেন্টে প্রকৃতপক্ষে পরিবর্তিত হয়েছে। এটি শোরগোল এবং কোটা ব্যবহার হ্রাস করে।

  • VCS (git diff) বা বিল্ড আর্টিফ্যাক্ট থেকে পরিবর্তিত পাথের তালিকা পান।
  • পাবলিক রুট দ্বারা ফিল্টার করুন (node_modules, .git, ব্যক্তিগত ফোল্ডারগুলি বাদ দিয়ে)।
  • অপ্রাসঙ্গিক URL পাঠানো এড়াতে ডোমেন/ভাষা অনুসারে গ্রুপ করুন।
প্রোডাকশনে একটি ফ্যালব্যাক থাকা দরকারী: যদি শুধুমাত্র-পরিবর্তিত খালি থাকে (উদাহরণস্বরূপ, কনফিগারেশনের জন্য হটফিক্স), গুরুত্বপূর্ণ সাইটম্যাপগুলিতে পিং করুন।

4) IndexNow বনাম সাইটম্যাপ পিং

  • গতি: IndexNow দ্রুত আপডেট সরবরাহ করে, ক্রল চক্রের জন্য অপেক্ষা করে না; সাইটম্যাপ পিং একটি সাইটম্যাপের উপস্থিতি নির্দেশ করে কিন্তু একটি নির্দিষ্ট URL-এ পরিবর্তন নয়।
  • শোরগোল: IndexNow - পরিবর্তিত URL-গুলিকে লক্ষ্য করে; সাইটম্যাপ - সম্পূর্ণ মানচিত্রের স্ক্যান (ক্রল বাজেটের জন্য বেশি ব্যয়বহুল)।
  • নির্ভরযোগ্যতা: সাইটম্যাপগুলি ভিত্তি হিসাবে থাকে; IndexNow সম্পূরক এবং দ্রুত করে। উভয় ব্যবহার করা ভাল।

5) সাধারণ ভুল

  • কীটি একটি পাবলিক URL এর মাধ্যমে অ্যাক্সেসযোগ্য নয় বা key প্যারামিটারের সাথে মেলে না।
  • ব্যক্তিগত/404 পৃষ্ঠাগুলি পিং করা, ক্যানোনিকালাইজেশন ছাড়া প্যারামিটার।
  • একই URL-গুলির খুব ঘন ঘন পুনরাবৃত্তি (পুনরাবৃত্তি থ্রোটল করুন)।
  • বহুভাষিকতা উপেক্ষা করা (সংশ্লিষ্ট স্থানীয়কৃত সংস্করণ পিং করতে হবে)।

6) প্রায়শই জিজ্ঞাসিত প্রশ্ন এবং চেকলিস্ট

একবারে কতগুলি URL পাঠাব?

একটি ব্যাচে 10,000 পর্যন্ত - সর্বোত্তমভাবে ধরন/ভাষা অনুসারে গ্রুপ করুন।

মুছে ফেলাগুলি পিং করব?

হ্যাঁ - ডিলিস্টিং দ্রুত করতে মুছে ফেলা পৃষ্ঠাগুলির URL পাস করুন।

কিভাবে নিশ্চিত করব যে এটি কাজ করছে?

API প্রতিক্রিয়া লগ (200/202), সার্ভার লগগুলিতে ক্রল স্পাইক, নতুন URL-গুলির দ্রুত উপস্থিতি।

  • কীটি স্থাপন করা হয়েছে এবং HTTPS এর মাধ্যমে অ্যাক্সেসযোগ্য।
  • CI/CD থেকে শুধুমাত্র-পরিবর্তিত পাথ সংগ্রহ করুন।
  • ডিডুপ্লিকেশন এবং পুনরাবৃত্তি থ্রোটলিং।
  • খালি ব্যাচে সাইটম্যাপ পিং-এ ফ্যালব্যাক।
: IndexNow নতুন এবং পরিবর্তিত পৃষ্ঠাগুলির ইন্ডেক্সিং উল্লেখযোগ্যভাবে ত্বরান্বিত করে। ঐতিহ্যগত সাইটম্যাপগুলির সাথে মিলিত হয়ে, এটি SEO অপ্টিমাইজেশনের জন্য একটি শক্তিশালী সরঞ্জাম।
