IndexNow: პრაქტიკული განხორციელება და რჩევები

15 სექტემბერი, 2025 • 5 წუთი საკითხავი
რატომ წაიკითხოთ როგორ სწრაფად მიაწოდოთ განახლებები საძიებო სისტემებს, როდის გამოიყენოთ მხოლოდ-შეცვლილი, როგორ განსხვავდება IndexNow sitemap ping-ისგან და როგორ თავიდან აიცილოთ ხშირი შეცდომები

1) რა არის IndexNow

IndexNow არის პროტოკოლი, რომელიც საშუალებას გაძლევთ სასწრაფოდ აცნობოთ საძიებო სისტემებს URL-ის ცვლილებების შესახებ (შექმნა, განახლება, წაშლა). მხარდაჭერილია Bing-ის, Yandex-ის, Seznam-ის და სხვა სერვისების მიერ. უპირატესობაა ინდექსირების მინიმალური დაგვიანება და ნაკლები crawl ხმაური.

2) გასაღების დაყენება და ძირითადი ping

  1. შექმენით გასაღები (GUID) და მოათავსეთ YOUR_KEY.txt ფაილი საიტის ფესვში (/.well-known/ ან დომენის ფესვში) გასაღების სტრიქონით:
  2. გაგზავნეთ ping ენდპოინტზე URL-ით და გასაღებით. მაგალითი (ერთი URL):
POST https://api.indexnow.org/indexnow Content-Type: application/json { "host": "www.serpmonn.ru", "key": "YOUR_KEY", "urlList": [ "https://www.serpmonn.ru/frontend/knowledge-base/articles/utm-complete-guide.html" ] }
ბაჩებისთვის მიუთითეთ ერთ მოთხოვნაში 10,000-მდე URL ან გამოიყენეთ ფაილის ფიდი ცვლილებების ლოგის ფაილის URL-ით:

3) მხოლოდ-შეცვლილი რეჟიმი

იდეა ის არის, რომ გააგზავნოთ ping მხოლოდ იმ ფაილებს, რომლებიც ბოლო კომიტზე/დეპლოიმენტზე რეალურად შეიცვალა. ეს ამცირებს ხმაურს და კვოტის გამოყენებას.

  • მიიღეთ შეცვლილი მარშრუტების სია VCS-დან (git diff) ან 빌д-ის არტეფაქტებიდან.
  • გაფილტრეთ საჯარო მარშრუტებით (გამორიცხეთ node_modules, .git, პირადი ფოლდერები).
  • დაჯგუფეთ დომენების/ენების მიხედვით, რათა თავიდან აიცილოთ შეუსაბამო URL-ების გაგზავნა.
პროდუქციაში სასარგებლოა ჰქონდეთ სარეზერვო ვარიანტი: თუ მხოლოდ-შეცვლილი ცარიელია (მაგალითად, კონფიგურაციებისთვის ცხელი შესწორება), გაგზავნეთ ping მნიშვნელოვან საიტის რუკებზე.

4) IndexNow vs sitemap ping

  • სიჩქარე: IndexNow უფრო სწრაფად აწვდის განახლებებს, არ ელოდება crawl ციკლს; sitemap ping აცნობებს საიტის რუკის არსებობას, მაგრამ არა კონკრეტული URL-ის ცვლილებებს.
  • ხმაური: IndexNow - მიზანმიმართული შეცვლილი URL-ებისთვის; sitemap - მთელი რუკის სკანირება (ძვირია crawl ბიუჯეტისთვის).
  • საიმედოობა: საიტის რუკები რჩება საფუძვლად; IndexNow ავსებს და აჩქარებს. უმჯობესია ორივე გამოიყენოთ.

5) ხშირი შეცდომები

  • გასაღები ხელმისაწვდომი არ არის საჯარო URL-ის მეშვეობით ან არ ემთხვევა key პარამეტრს.
  • პირად/404 გვერდებზე ping, პარამეტრები კანონიკიზაციის გარეშე.
  • იგივე URL-ების ძალიან ხშირი გამეორებები (შეაჩერეთ გამეორებები).
  • მრავალენოვნების იგნორირება (საჭიროა ping გაუგზავნოთ შესაბამის ლოკალიზებულ ვერსიას).

6) ხშირად დასმული კითხვები და საკონტროლო სია

რამდენი URL გამოგზავნოთ ერთდროულად

10,000-მდე ბაჩში - ოპტიმალურად დააჯგუფეთ ტიპის/ენის მიხედვით.

უნდა გამოგზავნოთ ping წაშლებისთვის

კი - გადაიტანეთ წაშლილი გვერდების URL-ები დეინდექსირების დასაჩქარებლად.

როგორ დავადასტუროთ, რომ მუშაობს

API პასუხის ლოგები (200/202), crawl-ის მწვერვალები სერვერის ლოგებში, ახალი URL-ების უფრო სწრაფი გამოჩენა.

  • გასაღები განთავსებულია და ხელმისაწვდომია HTTPS-ით.
  • მხოლოდ-შეცვლილი მარშრუტების შეგროვება CI/CD-დან.
  • დუბლიკატების აღმოფხვრა და გამეორებების შეჩერება.
  • სარეზერვო ping საიტის რუკაზე ცარიელი ბაჩის შემთხვევაში.
: IndexNow მნიშვნელოვნად აჩქარებს ახალი და შეცვლილი გვერდების ინდექსირებას. ტრადიციულ საიტის რუკებთან ერთად, ეს არის ძლიერი ინსტრუმენტი SEO ოპტიმიზაციისთვის.
