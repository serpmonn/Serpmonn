IndexNow: Implementasi Praktis dan Tips

15 September 2025 • 5 menit membaca
Mengapa membaca: bagaimana menyampaikan pembaruan dengan cepat ke mesin pencari, kapan menggunakan hanya-berubah, bagaimana IndexNow berbeda dari sitemap ping dan cara menghindari kesalahan umum.

1) Apa itu IndexNow

IndexNow adalah protokol yang memungkinkan Anda memberi tahu mesin pencari secara instan tentang perubahan URL (pembuatan, pembaruan, penghapusan). Didukung oleh Bing, Yandex, Seznam dan layanan lainnya. Keuntungannya adalah penundaan pengindeksan minimal dan lebih sedikit kebisingan crawl.

2) Pengaturan kunci dan ping dasar

  1. Buat kunci (GUID) dan letakkan file YOUR_KEY.txt di root situs (/.well-known/ atau root domain) dengan string kunci.
  2. Ping endpoint dengan URL dan kunci. Contoh (URL tunggal):
POST https://api.indexnow.org/indexnow Content-Type: application/json { "host": "www.serpmonn.ru", "key": "YOUR_KEY", "urlList": [ "https://www.serpmonn.ru/frontend/knowledge-base/articles/utm-complete-guide.html" ] }
Untuk batch, tentukan hingga 10.000 URL dalam satu permintaan atau gunakan feed file dengan URL file log perubahan.

3) Mode hanya-berubah

Ideanya adalah hanya melakukan ping pada file yang benar-benar berubah di komit/penyebaran terakhir. Ini mengurangi kebisingan dan penggunaan kuota.

  • Dapatkan daftar jalur yang diubah dari VCS (git diff) atau artefak build.
  • Filter dengan rute publik (kecuali node_modules, .git, folder pribadi).
  • Kelompokkan berdasarkan domain/bahasa untuk menghindari pengiriman URL yang tidak relevan.
Dalam produksi, berguna untuk memiliki fallback: jika hanya-berubah kosong (misalnya, perbaikan cepat untuk konfigurasi), ping sitemap penting.

4) IndexNow vs sitemap ping

  • Kecepatan: IndexNow memberikan pembaruan lebih cepat, tidak menunggu siklus crawl; sitemap ping menandakan kehadiran sitemap tetapi bukan perubahan pada URL tertentu.
  • Kebisingan: IndexNow - menargetkan URL yang diubah; sitemap - memindai seluruh peta (lebih mahal untuk anggaran crawl).
  • Keandalan: Sitemap tetap menjadi dasar; IndexNow melengkapi dan mempercepat. Lebih baik menggunakan keduanya.

5) Kesalahan umum

  • Kunci tidak dapat diakses melalui URL publik atau tidak cocok dengan parameter key.
  • Ping halaman pribadi/404, parameter tanpa kanonisasi.
  • Pengulangan terlalu sering dari URL yang sama (batasi pengulangan).
  • Mengabaikan multibahasa (harus ping versi terjemahan yang sesuai).

6) FAQ dan daftar periksa

Berapa banyak URL yang dikirim sekaligus?

Hingga 10.000 dalam satu batch - optimal dikelompokkan berdasarkan tipe/bahasa.

Haruskah ping penghapusan?

Ya - teruskan URL halaman yang dihapus untuk mempercepat deindexing.

Bagaimana memastikan itu berfungsi?

Log respons API (200/202), lonjakan crawl di log server, kemunculan URL baru lebih cepat.

  • Kunci ditempatkan dan dapat diakses melalui HTTPS.
  • Kumpulkan jalur hanya-berubah dari CI/CD.
  • Deduplikasi dan pembatasan pengulangan.
  • Fallback ke sitemap ping pada batch kosong.
: IndexNow secara signifikan mempercepat pengindeksan halaman baru dan yang diubah. Digabungkan dengan sitemap tradisional, ini adalah alat yang ampuh untuk optimasi SEO.
