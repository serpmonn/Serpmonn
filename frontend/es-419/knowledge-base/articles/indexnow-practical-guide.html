IndexNow: Implementación práctica y consejos

15 de septiembre de 2025 • 5 minutos de lectura

1) Qué es IndexNow

IndexNow es un protocolo que permite notificar instantáneamente a los motores de búsqueda sobre cambios en las URL (creación, actualización, eliminación). Compatible con Bing, Yandex, Seznam y otros servicios. La ventaja es un retraso mínimo de indexación y menos ruido de rastreo.

2) Configuración de clave y ping básico

  1. Genere una clave (GUID) y coloque el archivo YOUR_KEY.txt en la raíz del sitio (/.well-known/ o raíz del dominio) con la cadena de clave.
  2. Haga ping al endpoint con URL y clave. Ejemplo (URL única):
POST https://api.indexnow.org/indexnow Content-Type: application/json { "host": "www.serpmonn.ru", "key": "YOUR_KEY", "urlList": [ "https://www.serpmonn.ru/frontend/knowledge-base/articles/utm-complete-guide.html" ] }
Para lotes, especifique hasta 10.000 URL en una solicitud o use un feed de archivo con un archivo de registro de cambios.

3) Modo solo-cambiado

La idea es hacer ping solo a los archivos realmente cambiados en el último commit/implementación. Esto reduce el ruido y el uso de cuota.

  • Obtenga la lista de rutas cambiadas de VCS (git diff) o artefactos de compilación.
  • Filtre por rutas públicas (excluyendo node_modules, .git, carpetas privadas).
  • Agrupe por dominios/idiomas para evitar enviar URL irrelevantes.
En producción, es útil tener un respaldo: si solo-cambiado está vacío (por ejemplo, corrección de configuraciones), haga ping a sitemaps importantes.

4) IndexNow vs sitemap ping

  • Velocidad: IndexNow entrega actualizaciones más rápido, no espera el ciclo de rastreo; sitemap ping señala la presencia de un sitemap pero no cambios en URL específicas.
  • Ruido: IndexNow - dirigido a URL cambiadas; sitemap - escaneo de todo el mapa (más costoso para el presupuesto de rastreo).
  • Fiabilidad: Los sitemaps siguen siendo la base; IndexNow complementa y acelera. Es mejor usar ambos.

5) Errores comunes

  • Clave no accesible a través de URL pública o no coincide con el parámetro key.
  • Hacer ping a páginas privadas/404, parámetros sin canonicalización.
  • Repeticiones demasiado frecuentes de las mismas URL (limite las repeticiones).
  • Ignorar el multilingüismo (es necesario hacer ping a la versión localizada correspondiente).

6) Preguntas frecuentes y lista de verificación

¿Cuántas URL enviar a la vez?

Hasta 10.000 en un lote - agrupe de manera óptima por tipo/idioma.

¿Debe hacer ping a las eliminaciones?

Sí - pase las URL de páginas eliminadas para acelerar la desindexación.

¿Cómo confirmar que funciona?

Registros de respuesta de API (200/202), picos de rastreo en registros del servidor, aparición más rápida de nuevas URL.

  • Clave colocada y accesible a través de HTTPS.
  • Recolecte rutas solo-cambiadas desde CI/CD.
  • Deduplicación y limitación de repeticiones.
  • Respaldo a sitemap ping en lote vacío.
: IndexNow acelera significativamente la indexación de páginas nuevas y cambiadas. Combinado con sitemaps tradicionales, es una herramienta poderosa para la optimización SEO.
