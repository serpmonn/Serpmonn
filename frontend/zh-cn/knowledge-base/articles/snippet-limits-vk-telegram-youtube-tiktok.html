字符限制和片段截断 (VK / Telegram / YouTube / TikTok)

2025年9月15日
为什么阅读本文：确保标题和描述不被截断并提供最大CTR。以下 — 用于修正的限制汇总表和视觉“之前/之后”示例。

1) 限制和建议汇总表

注意：平台经常更改UI和截断方式。我们提供经过测试的推荐长度和已知的严格限制。

平台 标题（推荐/限制） 描述/文本（推荐/限制） 预览/截断
VK ≤ 60–70 / — ≤ 140–160 / 帖子中最多4096个字符 链接片段截断标题约70，描述约140+
Telegram ≤ 70 / —（从OG获取） ≤ 120–160 / 消息中4096 OG预览：标题约70，描述约120–140
YouTube ≤ 60 / 限制100 ≤ 150–200（前几行）/ 限制5000 Above‑the‑fold显示约2–3行
TikTok ≤ 60–80 / — ≤ 80–150（可见部分）/ 限制约2200 标题被截断，“更多”约80–150后

2) VK：“之前/之后”

之前

“关于我们新功能的极其详细且非常长的标题，将在所有预览中不可避免地截断并降低CTR…”

之后

“新功能：更快更简单” — 简短清晰。副标题/描述：“我们汇总了主要功能和用例”（≤ 140）。

3) Telegram：“之前/之后”

之前

“[新文章] 所有营销渠道UTM标签完整指南，含列表、示例、检查清单和目录”

之后

“UTM：完整指南” — “GA4、Google/Meta/VK模板、常见错误”。标题 ≤ 70，描述 ≤ 120–140。

4) YouTube：“之前/之后”

之前

“我们如何在30天内建立SEO：从审计到实施、分析和报告的所有步骤的完整分解 [2025]”（105字符）

之后

“30天SEO：分步计划” — “审计、实施、报告。错误和生活技巧（2025）”。标题 ≤ 60，描述的前150–200字符 — 关键好处和CTA。

5) TikTok：“之前/之后”

之前

“观看我们带有大量细节和链接的新长解释 — 记得读完！”

之后

“消耗流量的3个UTM错误” — “检查source/medium、编码和重定向。完整指南 — 通过链接”。

6) 预览和元数据提示

  • 在VK/Telegram上设置OG元标签（title/description/image）以获得正确的预览。
  • 保持title ≤ 60–70、description ≤ 140–160以获得普遍可见性。
  • 将主要好处放在描述的前1–2行。
  • 发布前检查预览（通过草稿/预览机器人/分享）。
