IndexNow: 实用实施与技巧

2025年9月15日 • 5分钟阅读
为什么阅读： 如何快速向搜索引擎交付更新、何时使用仅更改、IndexNow与站点地图ping有何不同以及如何避免常见错误

1) 什么是IndexNow

IndexNow是一种协议，允许您立即通知搜索引擎有关URL更改（创建、更新、删除）。由Bing、Yandex、Seznam和其他服务支持。优点是最小的索引延迟和更少的爬网噪音。

2) 密钥设置和基本ping

  1. 生成一个密钥（GUID）并将YOUR_KEY.txt文件放置于网站根目录（/.well-known/或域名根目录）并包含密钥字符串。
  2. 使用URL和密钥ping端点。示例（单个URL）：
POST https://api.indexnow.org/indexnow Content-Type: application/json { "host": "www.serpmonn.ru", "key": "YOUR_KEY", "urlList": [ "https://www.serpmonn.ru/frontend/knowledge-base/articles/utm-complete-guide.html" ] }
对于批次，在一个请求中指定最多10,000个URL，或使用带有更改日志文件URL的文件源。

3) 仅更改模式

想法是仅ping上次提交/部署中实际更改的文件。这减少了噪音和配额使用。

  • 从VCS（git diff）或构建工件获取已更改路径列表。
  • 按公共路由过滤（排除node_modules、.git、私有文件夹）。
  • 按域名/语言分组以避免发送不相关的URL。
在生产中，拥有回退机制是有用的：如果仅更改为空（例如，配置的热修复），则ping重要的站点地图。

4) IndexNow vs 站点地图ping

  • 速度: IndexNow更快地交付更新，不等待爬网周期；站点地图ping指示站点地图的存在，但非特定URL的更改。
  • 噪音: IndexNow - 针对已更改的URL；站点地图 - 扫描整个地图（对爬网预算更昂贵）。
  • 可靠性: 站点地图保持基础；IndexNow补充并加速。最好两者都使用。

5) 常见错误

  • 密钥无法通过公共URL访问或与key参数不匹配。
  • Ping私有/404页面，参数未经规范化。
  • 相同URL过于频繁的重复（限制重复）。
  • 忽略多语言（需要ping相应的本地化版本）。

6) 常见问题与检查清单

一次发送多少个URL？

一个批次中最多10,000个 - 按类型/语言最佳分组。

应该ping删除操作吗？

是的 - 传递已删除页面的URL以加速取消索引。

如何确认其正常工作？

API响应日志（200/202）、服务器日志中的爬网峰值、新URL更快出现。

  • 密钥已放置且可通过HTTPS访问。
  • 从CI/CD收集仅更改路径。
  • 去重和限制重复。
  • 空批次时回退到站点地图ping。
: IndexNow显著加速了新页面和已更改页面的索引。结合传统站点地图，这是一个强大的SEO优化工具。
