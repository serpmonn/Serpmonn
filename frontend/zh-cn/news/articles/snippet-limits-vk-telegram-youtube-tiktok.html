字符上限与摘要截断（VK / Telegram / YouTube / TikTok）

2025年9月15日
目标： 避免标题/描述被截断并提升 CTR。下方提供推荐长度表与前后对比示例。

1) 汇总表

平台 标题（推荐/上限） 描述/正文（推荐/上限） 预览/截断
VK ≤ 60–70 / — ≤ 140–160 / 最多 4096（帖子） 链接预览截断：标题 ~70，描述 ~140+
Telegram ≤ 70 / —（来自 OG） ≤ 120–160 / 消息 4096 OG 预览：标题 ~70，描述 ~120–140
YouTube ≤ 60 / 上限 100 ≤ 150–200（首屏）/ 上限 5000 首屏约 2–3 行可见
TikTok ≤ 60–80 / — ≤ 80–150（可见部分）/ 上限 ~2200 说明文字约 80–150 字后折叠

2) VK：前后对比

之前

“一个非常冗长的新功能标题，在所有预览中都会被截断，降低 CTR ……”

之后

“新功能：更快更简单” —— 简洁明确。副标题：“关键优势与示例”（≤ 140）。

3) Telegram：前后对比

之前

“【新文章】适用于所有营销渠道的 UTM 完整指南，含清单与示例”

之后

“UTM：完整指南” —— “GA4、Google/Meta/VK 模板、常见错误”。标题 ≤ 70，描述 ≤ 120–140。

4) YouTube：前后对比

之前

“我们如何在 30 天内搭建 SEO：从审计到上线、分析与报告的完整拆解【2025】”（105 字符）

之后

“30 天搞定 SEO：分步计划” —— “审计、上线、报告。坑与技巧（2025）”。标题 ≤ 60；描述前 150–200 字说明价值与 CTA。

5) TikTok：前后对比

之前

“看看我们很长的说明，包含大量细节与链接 —— 一定要看到最后！”

之后

“3 个吞流量的 UTM 错误” —— “检查 source/medium、编码与重定向。完整指南见链接”。

6) 预览与元数据建议

  • 为 VK/Telegram 设置正确的 OG meta（title/description/image）。
  • 标题 ≤ 60–70，描述 ≤ 140–160，跨平台更稳妥。
  • 把关键信息放在前 1–2 行。
  • 发布前用草稿/机器人/分享工具验证预览。
