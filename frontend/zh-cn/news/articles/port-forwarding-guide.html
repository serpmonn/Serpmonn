端口转发：2025 完整指南

2025年10月29日 • 15 分钟阅读

端口转发是家庭网络配置中对初学者最受需求同时也是最困难的主题之一。在本指南中，我们将详细讨论如何为任何目的正确设置端口转发：从游戏服务器到监控系统。

🔍 什么是端口转发？

端口转发 是一种技术，允许将网络请求从路由器的外部端口重定向到本地网络中设备的特定端口。

互联网

外部请求到达您的路由器

路由器

将请求重定向到所需设备

您的设备

接收并处理请求

⚠️ 重要须知

端口转发从互联网打开对您设备的访问权限。请始终遵循安全措施，不要打开不必要的端口！

🔧 设置准备

  1. 确定设备的静态 IP 地址

    您的设备（PC、服务器、摄像头）需要一个永久的本地 IP 地址，以便路由器知道将流量重定向到哪里。

    在 Windows 中如何查找 IP 地址：

    ipconfig

    查找您的网络连接的 "IPv4 地址" 行

    在 Linux/macOS 中如何查找 IP 地址：

    ifconfig

    或使用：ip addr show

  2. 配置静态 IP

    为避免 IP 地址更改，请静态配置：

    • 在路由器中：通过 DHCP 保留分配静态 IP
    • 在设备上：在网络设置中手动配置静态 IP

  3. 了解您的外部 IP 地址

    这是其他用户在互联网上看到的地址：

    查找外部 IP 的方法：

    # 通过命令行
curl ifconfig.me

# 或访问网站：
# - whatismyipaddress.com
# - 2ip.ru
# - myip.com

  4. 确定所需端口

    了解您的应用程序或服务使用哪些端口。此信息通常位于文档中。

🚀 分步设置说明

  1. 登录路由器控制面板

    打开浏览器并输入路由器的 IP 地址。通常为：

    • 192.168.1.1 (TP-Link, D-Link)
    • 192.168.0.1 (Netgear, 部分 TP-Link)
    • 192.168.31.1 (Xiaomi)
    • 192.168.10.1 (Huawei)

    默认用户名/密码通常列在路由器上的标签上。

  2. 查找端口转发部分

    该部分可能有不同的名称：

    • Port Forwarding
    • Virtual Servers
    • 端口转发
    • NAT
    • 防火墙

  3. 填写端口转发参数

    您需要指定：

    • 服务名称：任意名称（例如 "Minecraft 服务器"）
    • 外部端口：将在路由器上打开的端口
    • 内部端口：您设备上的端口
    • 内部 IP：您设备的静态 IP
    • 协议：TCP、UDP 或两者

  4. 保存设置

    填写所有字段后，按 "保存" 或 "应用"。路由器可能会重启。

📡 流行路由器的说明

🔄 TP-Link

  1. 登录控制面板 (192.168.1.1)
  2. 转到：高级 → NAT 转发 → 虚拟服务器
  3. 单击 "添加新"
  4. 填写：
    • 服务类型：从列表中选择或输入自定义
    • 外部端口：外部访问端口
    • 内部端口：设备上的端口
    • IP 地址：设备的本地 IP
    • 协议：TCP/UDP/BOTH
  5. 保存并重启路由器

🛡️ ASUS

  1. 登录面板 (192.168.1.1)
  2. 转到：高级设置 → 互联网 → 虚拟服务器/端口转发
  3. 启用 "启用端口转发"
  4. 填写字段：
    • 服务名称：服务名称
    • 端口范围：端口范围（例如 25565-25565）
    • 本地 IP：您设备的 IP
    • 本地端口：您设备的端口
    • 协议：TCP/UDP/BOTH
  5. 单击 "添加" 然后 "应用"

🔗 D-Link

  1. 登录面板 (192.168.0.1)
  2. 转到：高级 → 端口转发
  3. 单击 "添加"
  4. 填写：
    • 规则名称：规则名称
    • 外部端口：外部端口
    • 内部端口：内部端口
    • 内部 IP：设备 IP
    • 计划：始终
    • 入站过滤器：允许所有
  5. 保存设置

📶 Zyxel Keenetic

  1. 登录面板 (my.keenetic.net 或 192.168.1.1)
  2. 转到：安全 → 端口转发
  3. 单击 "添加规则"
  4. 填写：
    • 描述：服务名称
    • 接口：互联网
    • 协议：TCP/UDP
    • 外部端口：外部访问端口
    • 目标 IP：设备的本地 IP
    • 内部端口：设备上的端口
  5. 保存更改

✅ 验证端口转发工作

  1. 使用在线服务

    检查开放端口的网站：

    • portchecker.co
    • canyouseeme.org

  2. 通过命令行验证

    从另一台计算机检查：

    # 使用 telnet 或 nc (netcat) 
telnet 你的外部IP 端口 
# 或使用 nmap 
nmap -p 端口 你的外部IP

  3. 测试服务

    尝试从本地网络外的另一台设备（例如通过移动互联网）连接到您的服务。

✅ 设置成功的迹象

  • 端口在线检查器中显示为 "开放"
  • 您可以从外部连接到服务
  • 服务稳定运行无中断

🔧 常见问题故障排除

🚫 端口仍然关闭

  • 检查防火墙：在 Windows 防火墙或其他防火墙中添加例外
  • 重启路由器：有时设置仅在重启后应用
  • 检查 IP 地址：确保设备具有正确的静态 IP
  • 禁用 UPnP：有时 UPnP 与手动端口设置冲突

⚠️ 提供商的动态 IP

如果您有动态 IP，它可能会更改。解决方案：

  • 使用 DDNS：No-IP、DuckDNS 等服务
  • 检查 IP：定期更新有关当前 IP 的信息
  • 静态 IP：向提供商订购静态 IP（付费）

🔒 双重 NAT

当网络中有多个路由器时会出现问题：

  • 桥接模式：将其中一个路由器设置为桥接模式
  • DMZ：在主路由器上为第二个路由器配置 DMZ
  • 两者都转发：在两个路由器上配置转发

🛡️ 安全措施

🔐 关键规则

  1. 仅打开必要端口 - 每个开放端口都会增加攻击面
  2. 在所有可从外部访问的服务上使用复杂密码
  3. 定期更新软件 - 消除漏洞
  4. 使用路由器日志监控连接
  5. 考虑 VPN 作为端口转发的更安全替代方案

🎯 最佳实践

  • 使用非标准端口以减少扫描器的注意力
  • 根据时间表设置端口的自动禁用
  • 定期检查日志中的可疑活动
  • 使用 fail2ban 或类似工具阻止可疑 IP
