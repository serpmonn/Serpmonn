IndexNow：实用指南与技巧

2025年9月15日
为什么重要： 通过 IndexNow 将 URL 变更（新增/更新/删除）即时推送给搜索引擎，显著降低索引延迟与抓取噪声。

1) 什么是 IndexNow

IndexNow 是一种协议，可向搜索引擎主动告知 URL 的新增/更新/删除。目前由 Bing、Yandex、Seznam 等支持。它的优势是更快曝光变更，同时减少无效抓取。

2) 密钥验证与基础 ping

  1. 生成密钥（GUID），将 YOUR_KEY.txt 放置在站点根目录（/.well-known/ 或域名根）并写入密钥字符串。
  2. 向 endpoint 发送包含 URL 与密钥的请求。单 URL 示例：
POST https://api.indexnow.org/indexnow Content-Type: application/json { "host": "www.serpmonn.ru", "key": "YOUR_KEY", "urlList": [ "https://www.serpmonn.ru/frontend/zh-cn/news/articles/utm-complete-guide.html" ] }

批量发送时，可在单次请求中包含最多 10,000 个 URL，或提供一个变更文件的 URL。

3) changed‑only 模式

仅推送自上次提交/部署以来真正变动的 URL，以降低噪声与配额消耗。

  • 从 VCS（git diff）或构建产物中收集变更路径。
  • 仅保留公共路由（过滤 node_modules、.git、私有目录）。
  • 按域名/语言分组，避免推送不相关的 URL。
建议准备 fallback：当 changed‑only 为空（如仅改配置）时，回退到关键 sitemap 的 ping。

4) 与 sitemap ping 的对比

  • 速度： IndexNow 立即推送具体 URL；sitemap ping 仅提示地图有更新。
  • 噪声： IndexNow 精准；sitemap 可能触发更广泛抓取。
  • 可靠性： 二者结合最佳 —— sitemap 保覆盖，IndexNow 提即时性。

5) 常见错误

  • 密钥文件无法公开访问，或与请求中的 key 不一致。
  • 推送私有/404 页面，或使用未规范化的带参数 URL。
  • 同一 URL 重复推送频率过高（需要去重与限流）。
  • 忽略多语言页面（应分别推送本地化版本）。

6) FAQ 与上线清单

单批次可推送多少 URL？

最多 10,000 个，建议按类型/语言分组。

删除是否需要推送？

需要，可加速去索引。

如何验证生效？

查看 API 返回（200/202）、服务器抓取日志的波动，以及新 URL 的出现速度。

上线清单

  • 密钥已托管且可通过 HTTPS 访问。
  • CI/CD 集成 changed‑only 路径收集。
  • 重复去重与推送限流。
  • 当批次为空时回退到 sitemap ping。
