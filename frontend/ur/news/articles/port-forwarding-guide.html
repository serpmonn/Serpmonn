پورٹ فارورڈنگ: مکمل گائیڈ 2025

29 اکتوبر 2025 • 15 منٹ پڑھنے کا وقت

پورٹ فارورڈنگ گھریلو نیٹ ورک ترتیب میں نئے آنے والوں کے لیے سب سے زیادہ مانگے جانے والے اور ساتھ ہی سب سے مشکل موضوعات میں سے ایک ہے۔ اس گائیڈ میں، ہم تفصیل سے بات کریں گے کہ کسی بھی مقصد کے لیے پورٹ فارورڈنگ کو صحیح طریقے سے کیسے ترتیب دیا جائے: کھیل کے سرورز سے لے کر نگرانی کے نظام تک۔

🔍 پورٹ فارورڈنگ کیا ہے؟

پورٹ فارورڈنگ ایک ٹیکنالوجی ہے جو نیٹ ورک کی درخواستوں کو روٹر کے بیرونی پورٹ سے آپ کے مقامی نیٹ ورک میں موجود کسی آلے کے مخصوص پورٹ پر ری ڈائریکٹ کرنے کی اجازت دیتی ہے۔

⚠️ سمجھنا ضروری ہے

پورٹ فارورڈنگ انٹرنیٹ سے آپ کے آلے تک رسائی کھولتی ہے۔ ہمیشہ حفاظتی اقدامات پر عمل کریں اور غیر ضروری پورٹس مت کھولیں!

🔧 ترتیب کے لیے تیاری

  1. آلے کا مستقل IP پتہ مقرر کریں

    آپ کے آلے (PC, سرور, کیمرہ) کو ایک مستقل مقامی IP پتے کی ضرورت ہے تاکہ روٹر کو معلوم ہو کہ ٹریفک کہاں بھیجنی ہے۔

    ونڈوز میں IP پتہ کیسے معلوم کریں:

    ipconfig

    آپ کے نیٹ ورک کنکشن کے لیے "IPv4 Address" لائن تلاش کریں

    Linux/macOS میں IP پتہ کیسے معلوم کریں:

    ifconfig

    یا استعمال کریں: ip addr show

  2. مستقل IP ترتیب دیں

    IP پتے کے بدلنے سے بچنے کے لیے، اسے مستقل طور پر ترتیب دیں:

    • روٹر میں: DHCP ریزرویشن کے ذریعے مستقل IP مقرر کریں
    • آلے پر: نیٹ ورک کی ترتیبات میں دستی طور پر مستقل IP ترتیب دیں

  3. اپنا بیرونی IP پتہ معلوم کریں

    یہ وہ پتہ ہے جو دوسرے صارفین انٹرنیٹ پر دیکھتے ہیں:

    بیرونی IP معلوم کرنے کے طریقے:

    # کمانڈ لائن کے ذریعے
curl ifconfig.me

# یا ویب سائٹس ملاحظہ کریں:
# - whatismyipaddress.com
# - 2ip.ru
# - myip.com

  4. ضروری پورٹس مقرر کریں

    معلوم کریں کہ آپ کی ایپلیکیشن یا سروس کون سے پورٹس استعمال کرتی ہے۔ یہ معلومات عام طور پر دستاویزات میں ہوتی ہیں۔

🚀 مرحلہ وار ترتیب کی ہدایات

  1. روٹر کنٹرول پینل میں لاگ ان کریں

    براؤزر کھولیں اور روٹر کا IP پتہ درج کریں۔ عام طور پر یہ ہوتا ہے:

    • 192.168.1.1 (TP-Link, D-Link)
    • 192.168.0.1 (Netgear, کچھ TP-Link)
    • 192.168.31.1 (Xiaomi)
    • 192.168.10.1 (Huawei)

    صارف نام/پاس ورڈ عام طور پر روٹر پر اسٹیکر پر درج ہوتے ہیں۔

  2. پورٹ فارورڈنگ سیکشن تلاش کریں

    سیکشن کے مختلف نام ہو سکتے ہیں:

    • Port Forwarding
    • Virtual Servers
    • پورٹ فارورڈنگ
    • NAT
    • فائر وال

  3. پورٹ فارورڈنگ پیرامیٹرز بھریں

    آپ کو درج ذیل کی وضاحت کرنی ہوگی:

    • سروس کا نام: کوئی بھی نام (مثال کے طور پر "Minecraft سرور")
    • بیرونی پورٹ: روٹر پر کھلنے والا پورٹ
    • اندرونی پورٹ: آپ کے آلے پر پورٹ
    • اندرونی IP: آپ کے آلے کا مستقل IP
    • پروٹوکول: TCP, UDP یا دونوں

  4. ترتیبات محفوظ کریں

    تمام فیلڈز بھرنے کے بعد، "محفوظ کریں" یا "لاگو کریں" دبائیں۔ روٹر دوبارہ شروع ہو سکتا ہے۔

📡 مشہور روٹرز کے لیے ہدایات

🔄 TP-Link

  1. کنٹرول پینل میں لاگ ان کریں (192.168.1.1)
  2. اس پر جائیں: Advanced → NAT Forwarding → Virtual Servers
  3. "Add New" پر کلک کریں
  4. بھریں:
    • Service Type: فہرست سے منتخب کریں یا اپنا درج کریں
    • External Port: بیرونی رسائی کے لیے پورٹ
    • Internal Port: آلے پر پورٹ
    • IP Address: آلے کا مقامی IP
    • Protocol: TCP/UDP/BOTH
  5. محفوظ کریں اور روٹر دوبارہ شروع کریں

🛡️ ASUS

  1. پینل میں لاگ ان کریں (192.168.1.1)
  2. اس پر جائیں: Advanced Settings → WAN → Virtual Server/Port Forwarding
  3. "Enable Port Forwarding" کو فعال کریں
  4. فیلڈز بھریں:
    • Service Name: سروس کا نام
    • Port Range: پورٹ کی حد (مثال کے طور پر 25565-25565)
    • Local IP: آپ کے آلے کا IP
    • Local Port: آپ کے آلے کا پورٹ
    • Protocol: TCP/UDP/BOTH
  5. "Add" اور پھر "Apply" پر کلک کریں

🔗 D-Link

  1. پینل میں لاگ ان کریں (192.168.0.1)
  2. اس پر جائیں: Advanced → Port Forwarding
  3. "Add" پر کلک کریں
  4. بھریں:
    • Rule Name: قاعدہ کا نام
    • External Port: بیرونی پورٹ
    • Internal Port: اندرونی پورٹ
    • Internal IP: آلے کا IP
    • Schedule: Always
    • Inbound Filter: Allow All
  5. ترتیبات محفوظ کریں

📶 Zyxel Keenetic

  1. پینل میں لاگ ان کریں (my.keenetic.net یا 192.168.1.1)
  2. اس پر جائیں: سیکورٹی → پورٹ فارورڈنگ
  3. "قاعدہ شامل کریں" پر کلک کریں
  4. بھریں:
    • تفصیل: سروس کا نام
    • انٹرفیس: انٹرنیٹ
    • پروٹوکول: TCP/UDP
    • بیرونی پورٹ: بیرونی رسائی کے لیے پورٹ
    • ہدف IP: آلے کا مقامی IP
    • اندرونی پورٹ: آلے پر پورٹ
  5. تبدیلیاں محفوظ کریں

✅ پورٹ فارورڈنگ کے کام کی تصدیق

  1. آن لائن سروسز استعمال کریں

    کھلے پورٹس چیک کرنے کے لیے ویب سائٹس:

    • portchecker.co
    • canyouseeme.org

  2. کمانڈ لائن کے ذریعے تصدیق

    کسی دوسرے کمپیوٹر سے چیک کریں:

    # telnet یا nc (netcat) استعمال کریں 
telnet آپ-کا-بیرونی-IP پورٹ 
# یا nmap استعمال کرتے ہوئے 
nmap -p پورٹ آپ-کا-بیرونی-IP

  3. سروس ٹیسٹ

    اپنے مقامی نیٹ ورک سے باہر کسی دوسرے آلے سے (مثال کے طور پر موبائل انٹرنیٹ کے ذریعے) اپنی سروس سے رابطہ قائم کرنے کی کوشش کریں۔

✅ کامیاب ترتیب کی علامات

  • پورٹ آن لائن چیکرز میں "کھلا" کے طور پر ظاہر ہوتا ہے
  • آپ باہر سے سروس سے رابطہ کر سکتے ہیں
  • سروس بغیر رکاوٹ کے مستحکم طور پر چلتی ہے

🔧 عام مسائل کا حل

🚫 پورٹ اب بھی بند ہے

  • فائر وال چیک کریں: ونڈوز فائر وال یا کسی اور فائر وال میں استثنیٰ شامل کریں
  • روٹر دوبارہ شروع کریں: کبھی کبھار ترتیبات دوبارہ شروع کرنے کے بعد ہی لاگو ہوتی ہیں
  • IP پتہ چیک کریں: یقینی بنائیں کہ آلے کے پاس صحیح مستقل IP ہے
  • UPnP غیر فعال کریں: کبھی کبھار UPnP ہاتھ سے پورٹ کی ترتیبات سے متصادم ہوتا ہے

⚠️ فراہم کنندہ کا متحرک IP

اگر آپ کے پاس متحرک IP ہے، تو یہ بدل سکتا ہے۔ حل:

  • DDNS استعمال کریں: No-IP، DuckDNS جیسی خدمات
  • IP چیک کریں: موجودہ IP کے بارے میں معلومات باقاعدگی سے اپ ڈیٹ کریں
  • مستقل IP: فراہم کنندہ سے مستقل IP آرڈر کریں (ادا شدہ)

🔒 ڈبل NAT

مسئلہ اس وقت پیش آتا ہے جب آپ کے نیٹ ورک میں ایک سے زیادہ روٹرز ہوں:

  • برج موڈ: روٹرز میں سے ایک کو برج موڈ پر سیٹ کریں
  • DMZ: دوسرے روٹر کے لیے مرکزی روٹر پر DMZ ترتیب دیں
  • دونوں پر فارورڈنگ: دونوں روٹرز پر فارورڈنگ ترتیب دیں

🛡️ حفاظتی اقدامات

🔐 اہم قواعد

  1. صرف ضروری پورٹس کھولیں - ہر کھلا پورٹ حملے کی سطح بڑھاتا ہے
  2. بیرونی طور پر قابل رسائی تمام خدمات پر پیچیدہ پاس ورڈ استعمال کریں
  3. سافٹ ویئر کو باقاعدگی سے اپ ڈیٹ کریں - کمزوریوں کو دور کریں
  4. روٹر لاگز کا استعمال کرتے ہوئے کنکشنز کی نگرانی کریں
  5. پورٹ فارورڈنگ کے لیے زیادہ محفوظ متبادل کے طور پر VPN پر غور کریں

🎯 بہترین طریقے

  • اسکینرز کی توجہ کم کرنے کے لیے غیر معیاری پورٹس استعمال کریں
  • شیڈول کے مطابق پورٹس کی خودکار غیر فعالی ترتیب دیں
  • مشتبہ سرگرمی کے لیے لاگز کو باقاعدگی سے چیک کریں
  • مشتبہ IPs کو بلاک کرنے کے لیے fail2ban یا اسی طرح کا استعمال کریں
