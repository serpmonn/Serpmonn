محدودیت‌های کاراکتر و برش قطعه کد (VK / Telegram / YouTube / TikTok)

۲۴ شهریور ۱۴۰۴
چرا بخوانید: تا عنوان‌ها و توضیحات قطع نشوند و حداکثر CTR را ارائه دهند. در زیر — جدول خلاصه محدودیت‌ها و مثال‌های تصویری «قبل/بعد» برای اصلاح.

1) جدول خلاصه محدودیت‌ها و توصیه‌ها

توجه: پلتفرم‌ها اغلب UI و برش را تغییر می‌دهند. ما طول‌های توصیه شده تست شده و محدودیت‌های سخت شناخته شده را ارائه می‌دهیم.

پلتفرم عنوان (توصیه/محدودیت) توضیح/متن (توصیه/محدودیت) پیش‌نمایش/برش
VK ≤ ۶۰–۷۰ / — ≤ ۱۴۰–۱۶۰ / تا ۴۰۹۶ کاراکتر در پست قطعه کد لینک عنوان ~۷۰، توضیح ~۱۴۰+ را قطع می‌کند
Telegram ≤ ۷۰ / — (از OG گرفته شده) ≤ ۱۲۰–۱۶۰ / ۴۰۹۶ در پیام پیش‌نمایش از OG: عنوان ~۷۰، توضیح ~۱۲۰–۱۴۰
YouTube ≤ ۶۰ / محدودیت ۱۰۰ ≤ ۱۵۰–۲۰۰ (خطوط اول) / محدودیت ۵۰۰۰ Above‑the‑fold حدود ~۲–۳ خط نشان می‌دهد
TikTok ≤ ۶۰–۸۰ / — ≤ ۸۰–۱۵۰ (قسمت قابل مشاهده) / محدودیت تا ~۲۲۰۰ کپشن قطع می‌شود، «بیشتر» بعد از ~۸۰–۱۵۰

2) VK: «قبل/بعد»

قبل

«یک عنوان فوق‌العاده دقیق و بسیار طولانی درباره ویژگی جدید ما که به ناچار در همه پیش‌نمایش‌ها قطع خواهد شد و CTR را کاهش می‌دهد…»

بعد

«ویژگی جدید: سریع‌تر و ساده‌تر» — کوتاه و واضح. زیرعنوان/توضیح: «ما قابلیت‌های کلیدی و مثال‌های استفاده را جمع‌آوری کردیم» (≤ ۱۴۰).

3) Telegram: «قبل/بعد»

قبل

«[مقاله جدید] راهنمای کامل برچسب‌های UTM برای همه کانال‌های بازاریابی، با لیست‌ها، مثال‌ها، چک‌لیست و فهرست مطالب»

بعد

«UTM: راهنمای کامل» — «GA4، قالب‌های Google/Meta/VK، اشتباهات رایج». عنوان ≤ ۷۰، توضیح ≤ ۱۲۰–۱۴۰.

4) YouTube: «قبل/بعد»

قبل

«چگونه در ۳۰ روز SEO ساختیم: تجزیه کامل تمام مراحل از ممیزی تا اجرا، تحلیل و گزارش‌دهی [۲۰۲۵]» (۱۰۵ کاراکتر)

بعد

«SEO در ۳۰ روز: برنامه گام به گام» — «ممیزی، اجرا، گزارش‌دهی. اشتباهات و ترفندهای زندگی (۲۰۲۵)». عنوان ≤ ۶۰، ۱۵۰–۲۰۰ کاراکتر اول توضیح — مزایای کلیدی و CTA.

5) TikTok: «قبل/بعد»

قبل

«توضیح طولانی جدید ما را با جزئیات و لینک‌های زیاد تماشا کنید — حتماً تا انتها بخوانید!»

بعد

«۳ اشتباه UTM که ترافیک را می‌خورد» — «source/medium، رمزگذاری و تغییر مسیرها را بررسی کنید. راهنمای کامل — از طریق لینک».

6) نکات برای پیش‌نمایش‌ها و فراداده

  • برای پیش‌نمایش‌های صحیح در VK/Telegram، فراداده OG (title/description/image) را تنظیم کنید.
  • برای دید جهانی، title ≤ ۶۰–۷۰, description ≤ ۱۴۰–۱۶۰ نگه دارید.
  • مزیت کلیدی را در ۱–۲ خط اول توضیح قرار دهید.
  • قبل از انتشار، پیش‌نمایش‌ها را بررسی کنید (از طریق پیش‌نویس/بات‌های پیش‌نمایش/اشتراک‌گذاری).
