IndexNow: پیاده‌سازی عملی و نکات

۱) IndexNow چیست

IndexNow یک پروتکل است که به شما امکان می‌دهد موتورهای جستجو را به طور لحظه‌ای از تغییرات URL (ایجاد، به‌روزرسانی، حذف) مطلع کنید. توسط Bing، Yandex، Seznam و سایر سرویس‌ها پشتیبانی می‌شود. مزیت آن تأخیر حداقلی در ایندکس کردن و نویز کمتر کراول است.

۲) تنظیم کلید و پینگ پایه

  1. یک کلید (GUID) ایجاد کنید و فایل YOUR_KEY.txt را در ریشه سایت (/.well-known/ یا ریشه دامنه) با رشته کلید قرار دهید.
  2. به اندپوینت با URL و کلید پینگ بزنید. مثال (تک URL):
POST https://api.indexnow.org/indexnow Content-Type: application/json { "host": "www.serpmonn.ru", "key": "YOUR_KEY", "urlList": [ "https://www.serpmonn.ru/frontend/knowledge-base/articles/utm-complete-guide.html" ] }
برای دسته‌ها، تا ۱۰٬۰۰۰ URL را در یک درخواست مشخص کنید یا از فید فایل با URL فایل گزارش تغییرات استفاده کنید.

۳) حالت فقط-تغییر‌یافته

ایده این است که فقط به فایل‌هایی پینگ بزنید که در آخرین commit/استقرار واقعاً تغییر کرده‌اند. این نویز و استفاده از سهمیه را کاهش می‌دهد.

  • لیست مسیرهای تغییر یافته را از VCS (git diff) یا مصنوعات ساخت دریافت کنید.
  • با مسیرهای عمومی فیلتر کنید (به جز node_modules، .git، پوشه‌های خصوصی).
  • بر اساس دامنه/زبان گروه‌بندی کنید تا از ارسال URL نامربوط جلوگیری کنید.
در تولید، داشتن fallback مفید است: اگر فقط-تغییر‌یافته خالی باشد (مثلاً تعمیر سریع برای پیکربندی‌ها)، به نقشه‌های سایت مهم پینگ بزنید.

۴) IndexNow در مقابل sitemap ping

  • سرعت: IndexNow به‌روزرسانی‌ها را سریع‌تر تحویل می‌دهد، منتظر چرخه کراول نمی‌ماند؛ sitemap ping حضور یک نقشه سایت را نشان می‌دهد اما تغییرات در URL خاص را نه.
  • نویز: IndexNow - هدفمند روی URLهای تغییر یافته؛ sitemap - اسکن کل نقشه (گران‌تر برای بودجه کراول).
  • قابلیت اطمینان: نقشه‌های سایت پایه باقی می‌مانند؛ IndexNow تکمیل و تسریع می‌کند. بهتر است از هر دو استفاده کنید.

۵) خطاهای رایج

  • کلید از طریق URL عمومی قابل دسترسی نیست یا با پارامتر key مطابقت ندارد.
  • پینگ به صفحات خصوصی/404، پارامترها بدون متعارف‌سازی.
  • تکرارهای بیش از حد مکرر همان URLها (تکرارها را محدود کنید).
  • نادیده گرفتن چندزبانه بودن (نیاز به پینگ نسخه محلی شده مربوطه دارید).

۶) سوالات متداول و چک‌لیست

چند URL را همزمان ارسال کنم؟

تا ۱۰٬۰۰۰ در یک دسته - بهینه گروه‌بندی شده بر اساس نوع/زبان.

آیا باید حذف‌ها را پینگ کنم؟

بله - URL صفحات حذف شده را برای تسریع در حذف از ایندکس منتقل کنید.

چگونه تأیید کنم که کار می‌کند؟

لاگ‌های پاسخ API (200/202)، اسپایک‌های کراول در لاگ‌های سرور، ظهور سریع‌تر URLهای جدید.

  • کلید قرار داده شده و از طریق HTTPS قابل دسترسی است.
  • جمع‌آوری مسیرهای فقط-تغییر‌یافته از CI/CD.
  • حذف تکراری‌ها و محدود کردن تکرارها.
  • Fallback به sitemap ping در دسته خالی.
: IndexNow به طور قابل توجهی ایندکس کردن صفحات جدید و تغییر یافته را تسریع می‌کند. در ترکیب با نقشه‌های سایت سنتی، ابزار قدرتمندی برای بهینه‌سازی سئو است.
