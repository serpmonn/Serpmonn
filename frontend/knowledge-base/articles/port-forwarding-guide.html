Проброс портов на роутере: как открыть порты — инструкция 2026

16 июля 2026 • 15 минут чтения

Проброс портов (port forwarding) — настройка роутера, которая открывает внешний доступ к устройству в вашей сети: игровому серверу, ПК, камере или NAS. Ниже — как открыть порты на роутере по шагам, что такое внешний порт, как проверить переадресацию и что делать, если не работает. Инструкции для TP-Link, ASUS, D-Link и Zyxel Keenetic.

Что такое проброс портов?

Проброс портов (Port Forwarding) — это технология, которая позволяет перенаправлять сетевые запросы с внешнего порта роутера на конкретный порт устройства в вашей локальной сети.

Важно понимать

Проброс портов открывает доступ к вашему устройству из интернета. Всегда соблюдайте меры безопасности и не открывайте порты без необходимости!

Подготовка к настройке

  1. Определите статический IP-адрес устройства

    Вашему устройству (ПК, серверу, камере) нужен постоянный локальный IP-адрес, чтобы роутер знал, куда перенаправлять трафик.

    Как узнать IP-адрес в Windows:

    ipconfig

    Ищите строку "IPv4-адрес" для вашего сетевого подключения

    Как узнать IP-адрес в Linux/macOS:

    ifconfig

    Или используйте: ip addr show

  2. Настройте статический IP

    Чтобы IP-адрес не менялся, настройте его статически:

    • В роутере: Назначьте статический IP через DHCP-резервирование
    • На устройстве: Настройте статический IP вручную в сетевых настройках

  3. Узнайте внешний IP-адрес

    Это адрес, который видят другие пользователи в интернете:

    Способы узнать внешний IP:

    # Через командную строку
curl ifconfig.me

# Или посетите сайты:
# - whatismyipaddress.com
# - 2ip.ru
# - myip.com

  4. Определите нужные порты

    Узнайте, какие порты использует ваше приложение или сервис. Обычно эта информация есть в документации.

Пошаговая инструкция по настройке

  1. Войдите в панель управления роутером

    Откройте браузер и введите IP-адрес роутера. Обычно это:

    • 192.168.1.1 (TP-Link, D-Link)
    • 192.168.0.1 (Netgear, некоторые TP-Link)
    • 192.168.31.1 (Xiaomi)
    • 192.168.10.1 (Huawei)

    Логин/пароль по умолчанию обычно указаны на наклейке на роутере.

  2. Найдите раздел проброса портов

    Раздел может называться по-разному:

    • Port Forwarding
    • Виртуальные серверы (Virtual Servers)
    • Переадресация портов
    • NAT
    • Брандмауэр

  3. Заполните параметры проброса

    Вам потребуется указать:

    • Имя службы: Произвольное название (например, "Minecraft Server")
    • Внешний порт: Порт, который будет открыт на роутере
    • Внутренний порт: Порт на вашем устройстве
    • Внутренний IP: Статический IP вашего устройства
    • Протокол: TCP, UDP или Both (оба)

  4. Сохраните настройки

    После заполнения всех полей нажмите "Сохранить" или "Применить". Роутер может перезагрузиться.

Инструкции для популярных роутеров

TP-Link

  1. Войдите в панель управления (192.168.1.1)
  2. Перейдите: Advanced → NAT Forwarding → Virtual Servers
  3. Нажмите "Add New"
  4. Заполните:
    • Service Type: Выберите из списка или введите своё
    • External Port: Порт для доступа извне
    • Internal Port: Порт на устройстве
    • IP Address: Локальный IP устройства
    • Protocol: TCP/UDP/BOTH
  5. Сохраните и перезагрузите роутер

ASUS

  1. Войдите в панель (192.168.1.1)
  2. Перейдите: Advanced Settings → WAN → Virtual Server/Port Forwarding
  3. Включите "Enable Port Forwarding"
  4. Заполните поля:
    • Service Name: Название службы
    • Port Range: Диапазон портов (например, 25565-25565)
    • Local IP: IP вашего устройства
    • Local Port: Порт устройства
    • Protocol: TCP/UDP/BOTH
  5. Нажмите "Add" и "Apply"

D-Link

  1. Войдите в панель (192.168.0.1)
  2. Перейдите: Advanced → Port Forwarding
  3. Нажмите "Add"
  4. Заполните:
    • Rule Name: Имя правила
    • External Port: Внешний порт
    • Internal Port: Внутренний порт
    • Internal IP: IP устройства
    • Schedule: Always
    • Inbound Filter: Allow All
  5. Сохраните настройки

Zyxel Keenetic

  1. Войдите в панель Keenetic: my.keenetic.net или 192.168.1.1 (логин/пароль с корпуса)
  2. Новый интерфейс: Сетевые правила → Переадресация портов. Старый: Безопасность → Проброс портов
  3. Нажмите «Добавить правило» / «Создать»
  4. Укажите внешний порт (из интернета), внутренний порт (на ПК), IP устройства в LAN и протокол TCP/UDP
    • Описание: понятное имя (например Minecraft)
    • Интерфейс: Интернет / WAN
    • Протокол: TCP, UDP или оба — как требует сервис
    • Внешний порт: порт, который открываете снаружи
    • IP-адрес получателя: локальный IP ПК/сервера
    • Внутренний порт: порт службы на устройстве (часто совпадает с внешним)
  5. Сохраните правило. При смене локального IP настройте резервацию DHCP (статический аренд), иначе проброс «отвалится»
  6. Проверьте порт с мобильного интернета или через 2ip.ru / canyouseeme.org

Проверка работы проброса портов

  1. Используйте онлайн-сервисы

    Сайты для проверки открытых портов:

    • portchecker.co
    • canyouseeme.org
    • 2ip.ru/port-scan/

  2. Проверка через командную строку

    Проверка с другого компьютера:

    # Используйте telnet или nc (netcat) 
telnet ваш-внешний-ip порт 
# Или с помощью nmap 
nmap -p порт ваш-внешний-ip

  3. Тестирование сервиса

    Попробуйте подключиться к вашему сервису с другого устройства вне локальной сети (например через мобильный интернет).

Признаки успешной настройки

  • Порт показывает как "открытый" в онлайн-чекере
  • Вы можете подключиться к сервису извне
  • Сервис работает стабильно без обрывов

Решение распространенных проблем

Серый IP / CGNAT — порт всегда закрыт

Если роутер за CGNAT провайдера, классический проброс портов из интернета не заработает.

  • Проверьте тип IP у провайдера: нужен белый (публичный) IPv4
  • Сравните WAN IP роутера с IP на 2ip.ru — если разные, скорее всего CGNAT
  • Закажите белый IP или используйте VPN/туннель с port forward
  • Не тратьте время на бесконечные перезагрузки роутера, пока нет белого IP

Порт все еще закрыт

  • Проверьте брандмауэр: Добавьте исключение в Windows Firewall или другой файрвол
  • Перезагрузите роутер: Иногда настройки применяются только после перезагрузки
  • Проверьте IP-адрес: Убедитесь, что устройство имеет правильный статический IP
  • Отключите UPnP: Иногда UPnP конфликтует с ручной настройкой портов

Динамический IP провайдера

Если у вас динамический IP, он может меняться. Решения:

  • Используйте DDNS: Сервисы типа No-IP, DuckDNS
  • Проверяйте IP: Регулярно обновляйте информацию о текущем IP
  • Статический IP: Закажите у провайдера статический IP (платно)

Двойной NAT

Проблема возникает, когда у вас несколько роутеров в сети:

  • Режим моста: Настройте один из роутеров в режим моста
  • DMZ: Настройте DMZ на основном роутере для второго роутера
  • Проброс на обоих: Настройте проброс на обоих роутерах

Меры безопасности

Критически важные правила

  1. Открывайте только необходимые порты - каждый открытый порт увеличивает поверхность атаки
  2. Используйте сложные пароли на всех сервисах, доступных извне
  3. Регулярно обновляйте ПО - устраняйте уязвимости
  4. Отслеживайте подключения с помощью логов роутера
  5. Рассмотрите VPN как более безопасную альтернативу пробросу портов

Лучшие практики

  • Используйте нестандартные порты для снижения внимания сканеров
  • Настройте автоматическое отключение портов по расписанию
  • Регулярно проверяйте логи на предмет подозрительной активности
  • Используйте fail2ban или аналоги для блокировки подозрительных IP

Практика: частые сценарии проброса портов

Ниже — готовые схемы под запросы, по которым чаще всего ищут инструкцию: внешний порт, Minecraft (25565), два роутера и CGNAT.

Что такое внешний порт

Внешний порт — номер, на который приходит соединение из интернета на WAN-IP роутера. Внутренний порт — куда роутер перенаправит это соединение в LAN. Часто они совпадают (например 25565→25565), но можно открыть снаружи 25565, а на ПК слушать другой порт.

Minecraft / порт 25565

Для Java-сервера Minecraft обычно открывают TCP 25565. На ПК должен быть запущен сервер, firewall Windows — разрешить входящие на 25565, в роутере — правило port forwarding на IP компьютера с сервером. Подключаются друзья по вашему белому IP (или DDNS): ваш-ip:25565.

Как открыть порты при двух роутерах

Проброс нужен на каждом NAT-е по пути. Чаще: на первом (от провайдера) проброс на IP второго роутера, на втором — на ПК. Либо переведите второй роутер в режим AP/bridge, чтобы остался один NAT.

CGNAT: почему порт «не открывается»

Если провайдер выдаёт серый IP (CGNAT), с интернета до вашего роутера «не достучаться» даже при идеальном пробросе. Признаки: в кабинете оператора нет белого IP, онлайн-чекер всегда Closed. Решение: заказать белый IP, IPv6 (если сервис умеет) или туннель/VPN с пробросом.

Частые вопросы о пробросе портов

Что такое проброс портов на роутере?

Проброс портов (переадресация, port forwarding) направляет входящий трафик с внешнего IP роутера на выбранный компьютер или устройство во внутренней сети по указанному порту.

Как открыть порты на роутере?

Зайдите в веб-панель роутера (часто 192.168.0.1 или 192.168.1.1), найдите раздел Virtual Server / Port Forwarding / Переадресация портов, укажите внешний и внутренний порт, IP устройства в LAN и сохраните правило. На устройствах с CGNAT у провайдера проброс может не работать — тогда нужен белый IP или VPN.

Для чего пробрасывать порты?

Чаще всего — для игр (в том числе порт 25565 для Minecraft), своего сервера, удалённого рабочего стола, видеонаблюдения и доступа к файлам из интернета.

Как проверить, что проброс портов работает?

Убедитесь, что на устройстве запущена служба на нужном порту, затем проверьте доступ с внешней сети или через онлайн-проверка порта. В гайде ниже есть раздел проверки и типичные ошибки.

Keenetic: как сделать проброс порта?

В интерфейсе Keenetic откройте «Сетевые правила» → «Переадресация портов», добавьте правило с портом и устройством. Подробные шаги — в разделе инструкций по роутерам на этой странице.

Какой порт пробросить для Minecraft?

По умолчанию для Java Edition — TCP 25565 на компьютер, где запущен сервер. UDP нужен только если этого требует конкретный мод/прокси; для классического сервера достаточно TCP.

Что такое внешний порт для роутера?

Это порт на стороне интернета (WAN). Клиент подключается к вашему публичному IP:внешний_порт, а роутер перенаправляет пакет на внутренний IP:внутренний_порт.

Как проверить, работает ли переадресация портов?

Запустите службу на ПК, отключите блокировку firewall для этого порта, откройте 2ip.ru/canyouseeme с мобильного интернета (не с Wi‑Fi дома) и просканируйте порт. Open/успех = правило работает.

Почему не работает проброс портов на Keenetic?

Частые причины: неверный LAN‑IP (нет DHCP‑reserv), правило на другой интерфейс, firewall на ПК, CGNAT у провайдера, либо смотрите не тот раздел меню (в новых прошивках — «Сетевые правила → Переадресация портов»).
