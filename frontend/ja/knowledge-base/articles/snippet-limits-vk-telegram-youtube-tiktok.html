文字数制限とスニペット切り捨て (VK / Telegram / YouTube / TikTok)

2025年9月15日
なぜ読むべきか: タイトルと説明が切り取られず、最大CTRを提供するため。以下 — 修正のための制限の要約表と視覚的な「前/後」の例。

1) 制限と推奨事項の要約表

注記: プラットフォームは頻繁にUIと切り捨てを変更します。テスト済みの推奨長さと既知の厳しい制限を提供します。

プラットフォーム タイトル (推奨/制限) 説明/テキスト (推奨/制限) プレビュー/切り捨て
VK ≤ 60–70 / — ≤ 140–160 / 投稿内で最大4096文字 リンクスニペットはタイトルを約70、説明を約140+切り捨てます
Telegram ≤ 70 / — (OGから取得) ≤ 120–160 / メッセージ内で4096 OGからのプレビュー: タイトル約70、説明約120–140
YouTube ≤ 60 / 制限100 ≤ 150–200 (最初の行) / 制限5000 Above‑the‑foldは約2–3行を表示
TikTok ≤ 60–80 / — ≤ 80–150 (表示部分) / 制限約2200まで キャプションが切り捨てられ、「もっと」約80–150後に表示

2) VK: 「前/後」

前

「すべてのプレビューで必然的に切り取られ、CTRを低下させる新機能に関する信じられないほど詳細で非常に長いタイトル…」

後

「新機能: より速く、より簡単」 — 短く明確です。サブタイトル/説明: 「主要な機能と使用例を集めました」 (≤ 140)。

3) Telegram: 「前/後」

前

「[新記事] すべてのマーケティングチャネル向けUTMタグの完全ガイド、リスト、例、チェックリスト、目次付き」

後

「UTM: 完全ガイド」 — 「GA4、Google/Meta/VK向けテンプレート、一般的な間違い」。タイトル ≤ 70、説明 ≤ 120–140。

4) YouTube: 「前/後」

前

「30日間でSEOを構築した方法: 監査から実装、分析、レポーティングまでの全ステップの完全な内訳 [2025]」 (105文字)

後

「30日間でSEO: ステップバイステップ計画」 — 「監査、実装、レポーティング。間違いとライフハック (2025)」。タイトル ≤ 60、説明の最初の150–200文字 — 主要な利点とCTA。

5) TikTok: 「前/後」

前

「多くの詳細とリンクを含む新しい長い説明を視聴 — 最後まで読むことを忘れずに！」

後

「トラフィックを消費する3つのUTMミス」 — 「source/medium、エンコーディング、リダイレクトを確認してください。完全ガイド — リンク経由」。

6) プレビューとメタデータのヒント

  • VK/Telegramで正しいプレビューのためにOGメタ (title/description/image)を設定します。
  • ユニバーサルな可視性のためにtitle ≤ 60–70、description ≤ 140–160を維持します。
  • 説明の最初の1–2行に主要な利点を配置します。
  • 公開前にプレビューを確認します (下書き/プレビューボット/共有を通じて)。
