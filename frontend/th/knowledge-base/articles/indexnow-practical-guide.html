IndexNow: การนำไปใช้จริงและเคล็ดลับ

15 กันยายน 2025 • 5 นาทีในการอ่าน
เหตุผลที่ควรอ่าน: วิธีการส่งมอบการอัปเดตไปยังเสิร์ชเอนจิ้นอย่างรวดเร็ว, เมื่อใดที่ควรใช้เฉพาะที่เปลี่ยนแปลง, IndexNow แตกต่างจาก sitemap ping อย่างไร และวิธีหลีกเลี่ยงข้อผิดพลาดทั่วไป

1) IndexNow คืออะไร

IndexNow เป็นโปรโตคอลที่ช่วยให้คุณสามารถแจ้งเสิร์ชเอนจิ้นทันทีเกี่ยวกับการเปลี่ยนแปลงของ URL (การสร้าง, การอัปเดต, การลบ) รองรับโดย Bing, Yandex, Seznam และบริการอื่นๆ ข้อดีคือความล่าช้าในการจัดทำดัชนีน้อยที่สุดและมีเสียงรบกวนจากการครอว์ลน้อยลง

2) การตั้งค่าคีย์และการ ping พื้นฐาน

  1. สร้างคีย์ (GUID) และวางไฟล์ YOUR_KEY.txt ในรูทของไซต์ (/.well-known/ หรือรูทโดเมน) พร้อมกับสตริงคีย์
  2. ping ไปยัง endpoint ด้วย URL และคีย์ ตัวอย่าง (URL เดียว):
POST https://api.indexnow.org/indexnow Content-Type: application/json { "host": "www.serpmonn.ru", "key": "YOUR_KEY", "urlList": [ "https://www.serpmonn.ru/frontend/knowledge-base/articles/utm-complete-guide.html" ] }
สำหรับแบทช์ ระบุ URL ได้สูงสุด 10,000 รายการในการร้องขอหนึ่งครั้ง หรือใช้ฟีดไฟล์ที่มี URL ของไฟล์บันทึกการเปลี่ยนแปลง

3) โหมดเฉพาะที่เปลี่ยนแปลง

แนวคิดคือ ping เฉพาะไฟล์ที่เปลี่ยนแปลงจริงในการคอมมิต/ดีพลอยครั้งล่าสุด สิ่งนี้ช่วยลดเสียงรบกวนและการใช้โควต้า

  • รับรายการเส้นทางที่เปลี่ยนแปลงจาก VCS (git diff) หรือ artifacts การสร้าง
  • กรองตามเส้นทางสาธารณะ (ไม่รวม node_modules, .git, โฟลเดอร์ส่วนตัว)
  • จัดกลุ่มตามโดเมน/ภาษาเพื่อหลีกเลี่ยงการส่ง URL ที่ไม่เกี่ยวข้อง
ในสภาพแวดล้อมการผลิต มีประโยชน์ที่จะมี fallback: หากเฉพาะที่เปลี่ยนแปลงว่างเปล่า (เช่น การแก้ไขด่วนสำหรับการตั้งค่า) ให้ ping แผนผังเว็บไซต์ที่สำคัญ

4) IndexNow กับ sitemap ping

  • ความเร็ว: IndexNow ส่งมอบการอัปเดตได้เร็วกว่า ไม่รอรอบการครอว์ล; sitemap ping บ่งบอกถึงการมีอยู่ของแผนผังเว็บไซต์ แต่ไม่ใช่การเปลี่ยนแปลงของ URL เฉพาะ
  • เสียงรบกวน: IndexNow - กำหนดเป้าหมายไปที่ URL ที่เปลี่ยนแปลง; sitemap - การสแกนแผนที่ทั้งหมด (ใช้งบประมาณการครอว์ลมากขึ้น)
  • ความน่าเชื่อถือ: แผนผังเว็บไซต์ยังคงเป็นพื้นฐาน; IndexNow เสริมและเร่งความเร็ว ควรใช้ทั้งสองอย่าง

5) ข้อผิดพลาดทั่วไป

  • คีย์ไม่สามารถเข้าถึงได้ผ่าน URL สาธารณะหรือไม่ตรงกับพารามิเตอร์ key
  • ping หน้าเว็บส่วนตัว/404 พารามิเตอร์โดยไม่มีการ canonicalization
  • การทำซ้ำบ่อยเกินไปของ URL เดียวกัน (จำกัดการทำซ้ำ)
  • ละเลยหลายภาษา (ต้อง ping เวอร์ชันที่แปลแล้วที่เกี่ยวข้อง)

6) คำถามที่พบบ่อยและรายการตรวจสอบ

ควรส่ง URL กี่รายการในครั้งเดียว?

สูงสุด 10,000 ในแบทช์ - จัดกลุ่มตามประเภท/ภาษาอย่างเหมาะสม

ควร ping การลบหรือไม่?

ใช่ - ส่ง URL ของหน้าที่ลบแล้วเพื่อเร่งการลบออกจากดัชนี

ยืนยันได้อย่างไรว่าทำงาน?

บันทึกการตอบสนอง API (200/202), การเพิ่มขึ้นของการครอว์ลในบันทึกเซิร์ฟเวอร์, URL ใหม่ปรากฏเร็วขึ้น

  • คีย์ถูกวางและเข้าถึงได้ผ่าน HTTPS
  • รวบรวมเส้นทางเฉพาะที่เปลี่ยนแปลงจาก CI/CD
  • การกำจัดการซ้ำซ้อนและการจำกัดการทำซ้ำ
  • Fallback ไปที่ sitemap ping เมื่อแบทช์ว่างเปล่า
: IndexNow เร่งการจัดทำดัชนีของหน้าใหม่และหน้าที่เปลี่ยนแปลงได้อย่างมาก เมื่อรวมกับแผนผังเว็บไซต์แบบดั้งเดิมแล้ว มันเป็นเครื่องมือที่มีประสิทธิภาพสำหรับการปรับแต่ง SEO
