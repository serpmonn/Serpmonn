ข้อจำกัดอักขระและการตัดข้อความสั้น (VK / Telegram / YouTube / TikTok)

15 กันยายน 2025
ทำไมต้องอ่าน: เพื่อให้แน่ใจว่าชื่อเรื่องและคำอธิบายไม่ถูกตัดและให้ CTR สูงสุด ด้านล่าง — ตารางสรุปข้อจำกัดและตัวอย่างภาพ 'ก่อน/หลัง' สำหรับการแก้ไข

1) ตารางสรุปข้อจำกัดและข้อแนะนำ

หมายเหตุ: แพลตฟอร์มมักเปลี่ยน UI และการตัด เราให้ความยาว ที่แนะนำ ที่ทดสอบแล้วและข้อจำกัดเข้มที่รู้จัก

แพลตฟอร์ม ชื่อเรื่อง (แนะนำ/จำกัด) คำอธิบาย/ข้อความ (แนะนำ/จำกัด) แสดงตัวอย่าง/ตัด
VK ≤ 60–70 / — ≤ 140–160 / สูงสุด 4096 ตัวอักษรในโพสต์ สnippet ลิงก์ตัดชื่อเรื่อง ~70 คำอธิบาย ~140+
Telegram ≤ 70 / — (นำมาจาก OG) ≤ 120–160 / 4096 ในข้อความ แสดงตัวอย่างจาก OG: ชื่อเรื่อง ~70 คำอธิบาย ~120–140
YouTube ≤ 60 / จำกัด 100 ≤ 150–200 (บรรทัดแรก) / จำกัด 5000 Above‑the‑fold แสดง ~2–3 บรรทัด
TikTok ≤ 60–80 / — ≤ 80–150 (ส่วนที่มองเห็น) / จำกัดสูงสุด ~2200 คำบรรยายถูกตัด 'เพิ่มเติม' หลัง ~80–150

2) VK: 'ก่อน/หลัง'

ก่อน

"ชื่อเรื่องที่ละเอียดอย่างไม่น่าเชื่อและยาวมากเกี่ยวกับฟีเจอร์ใหม่ของเราซึ่งจะถูกตัดในตัวอย่างการแสดงตัวอย่างทั้งหมดและลด CTR อย่างหลีกเลี่ยงไม่ได้..."

หลัง

„ฟีเจอร์ใหม่: เร็วและง่ายขึ้น” — สั้นและชัดเจน คำบรรยาย/คำอธิบาย: „เราได้รวบรวมความสามารถหลักและตัวอย่างการใช้งาน” (≤ 140).

3) Telegram: 'ก่อน/หลัง'

ก่อน

"[บทความใหม่] คู่มือฉบับสมบูรณ์สำหรับแท็ก UTM สำหรับทุกช่องทางการตลาด พร้อมรายการ ตัวอย่าง รายการตรวจสอบและสารบัญ"

หลัง

„UTM: คู่มือฉบับสมบูรณ์” — „GA4 เทมเพลตสำหรับ Google/Meta/VK ข้อผิดพลาดทั่วไป” ชื่อเรื่อง ≤ 70 คำอธิบาย ≤ 120–140.

4) YouTube: 'ก่อน/หลัง'

ก่อน

"วิธีที่เราสร้าง SEO ใน 30 วัน: การแจกแจงขั้นตอนทั้งหมดตั้งแต่การตรวจสอบไปจนถึงการใช้งาน การวิเคราะห์และการรายงาน [2025]" (105 ตัวอักษร)

หลัง

„SEO ใน 30 วัน: แผนทีละขั้นตอน” — „การตรวจสอบ การใช้งาน การรายงาน ข้อผิดพลาดและเคล็ดลับชีวิต (2025)” ชื่อเรื่อง ≤ 60 150–200 ตัวอักษรแรกของคำอธิบาย — ประโยชน์หลักและ CTA

5) TikTok: 'ก่อน/หลัง'

ก่อน

"ดูคำอธิบายยาวใหม่ของเราพร้อมรายละเอียดและลิงก์มากมาย — อย่าลืมอ่านจนจบ!"

หลัง

„3 ข้อผิดพลาด UTM ที่กลืนการเข้าชม” — „ตรวจสอบ source/medium การเข้ารหัสและการเปลี่ยนเส้นทาง คู่มือฉบับสมบูรณ์ — ผ่านลิงก์".

6) เคล็ดลับสำหรับการแสดงตัวอย่างและเมตาดาต้า

  • ตั้งค่า เมตา OG (title/description/image) สำหรับการแสดงตัวอย่างที่ถูกต้องใน VK/Telegram
  • รักษา title ≤ 60–70, description ≤ 140–160 สำหรับการมองเห็นสากล
  • วาง ประโยชน์หลักใน 1–2 บรรทัดแรก ของคำอธิบาย
  • ตรวจสอบตัวอย่างก่อนเผยแพร่ (ผ่านแบบร่าง/บอทแสดงตัวอย่าง/การแชร์)
