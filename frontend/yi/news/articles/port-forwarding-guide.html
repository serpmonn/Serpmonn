פּאָרט פאָרווערדינג: פול גייד 2025

פּאָרט פאָרווערדינג איז איינע פֿון די מערסט געבעטן און גלייך מערסט קאָמפּלעקס טעמעס פֿאַר נוביצע אין שטעלן היים נעץ. אין דעם גייד, מיר דערקלערן אין דעטאַל ווי צו שטעלן פּאָרט פאָרווערדינג ריכטיק פֿאַר קיין ציל: פֿון שפּיל סערווערס ביז איבערוואַך סיסטעמען.

🔍 וואָס איז פּאָרט פאָרווערדינג?

⚠️ וויכטיק צו פֿאַרשטיין

פּאָרט פאָרווערדינג עפנט צוטריט צו דיין מיטל פֿון אינטערנעט. נאָכגיין זיכערהייט מיטלען אַלע מאָל און עפענען ניט ניט-דארף פּאָרץ!

🔧 צוגרייטונג פֿאַר שטעלן

  1. באַשטימען מיטלס שטעטיק IP אַדרעס

    דיין מיטל (קאָמפּיוטער, סערווער, קאַמעראַ) דאַרף אַ שטעטיק לאָקאַל IP אַדרעס פֿאַר דעם ראַוטער צו וויסן וואוהין צו ווייטערשיקן די טראַפיק.

    ווי צו געפֿינען IP אַדרעס אין Windows:

    ipconfig

    זוך די "IPv4 Address" שורה פֿאַר דיין נעץ פֿאַרבינדונג

    ווי צו געפֿינען IP אַדרעס אין Linux/macOS:

    ifconfig

    אָדער נוץ: ip addr show

  2. שטעלן שטעטיק IP

    צו פֿאַרהיטן IP אַדרעס ענדערונגען, שטעל עס שטעטיק:

    • אין ראַוטער: שטעל שטעטיק IP דורך DHCP רעזערוואַציע
    • אויף מיטל: שטעל שטעטיק IP מאַנואַלי אין נעץ סעטטינגס

  3. וויסן דיין אויסערדיקע IP אַדרעס

    דאָס איז די אַדרעס וואָס אנדערע באַניצער זען אויף אינטערנעט:

    וועגן צו וויסן אויסערדיקע IP:

    # דורך באַפֿעל שורה
curl ifconfig.me

# אָדער באַזוך וועבזייטלען:
# - whatismyipaddress.com
# - 2ip.ru
# - myip.com

  4. באַשטימען נייטיקע פּאָרץ

    וויסן וועלכע פּאָרץ דיין אַפּלאַקיישאַן אָדער דינסט ניצט. די אינפֿאָרמאַציע איז געווענלעך אין דאָקומענטן.

🚀 שריט-דורך-שריט שטעלן אינסטרוקציע

  1. קלאָגן אַריין אין ראַוטער קאָנטראָל פּאַנעל

    עפֿן דיין וועב בלעטערער און טיפּ די ראַוטערס IP אַדרעס. דאָס איז געווענלעך:

    • 192.168.1.1 (TP-Link, D-Link)
    • 192.168.0.1 (Netgear, עטלעכע TP-Link)
    • 192.168.31.1 (Xiaomi)
    • 192.168.10.1 (Huawei)

    סטאַנדאַרט באַניצער נאָמען/פּאַראָל איז געווענלעך אויף די ראַוטער שטעקן.

  2. געפֿינען פּאָרט פאָרווערדינג אָפּטיילונג

    דער אָפּטייל קען האָבן פאַרשידענע נעמען:

    • Port Forwarding
    • Virtual Servers
    • פּאָרט פאָרווערדינג
    • NAT
    • פֿייערוואָל

  3. פֿילן פּאָרט פאָרווערדינג פּאַראַמעטערס

    דו מוזן באַשטימען:

    • דינסט נאָמען: קיין נאָמען (צ.ב.ש "מייניקראַפט סערווער")
    • אויסערדיקער פּאָרט: פּאָרט געעפֿנט אויף ראַוטער
    • אינערלעכער פּאָרט: פּאָרט אויף דיין מיטל
    • אינערלעכע IP: דיין מיטלס שטעטיק IP
    • פּראָטאָקאָל: TCP, UDP אָדער ביידע

  4. היט סעטטינגס

    נאָך פֿילן אַלע פעלדער, דריקט אויף "היט" אָדער "אויספֿירן". דער ראַוטער קען רעבאָאָט.

📡 אינסטרוקציע פֿאַר פּאָפּולאַרע ראַוטערס

🔄 TP-Link

  1. קלאָגן אַריין אין קאָנטראָל פּאַנעל (192.168.1.1)
  2. גיי צו: אַוואַנסירט → NAT פאָרווערדינג → ווירטואַל סערווערס
  3. דריק "Add New"
  4. פֿילן:
    • Service Type: קלויב פֿון ליסטע אָדער טיפּ זיך
    • External Port: פּאָרט פֿאַר אויסערדיקע צוטריט
    • Internal Port: פּאָרט אויף מיטל
    • IP Address: מיטלס לאָקאַל IP
    • Protocol: TCP/UDP/BOTH
  5. היט און רעבאָאָט ראַוטער

🛡️ ASUS

  1. קלאָגן אַריין אין פּאַנעל (192.168.1.1)
  2. גיי צו: אַוואַנסירטע סעטטינגס → WAN → ווירטואַל סערווער/פּאָרט פאָרווערדינג
  3. אַקטיווירן "Enable Port Forwarding"
  4. פֿילן פעלדער:
    • Service Name: דינסט נאָמען
    • Port Range: פּאָרט קייט (צ.ב.ש 25565-25565)
    • Local IP: דיין מיטלס IP
    • Local Port: דיין מיטלס פּאָרט
    • Protocol: TCP/UDP/BOTH
  5. דריק "Add" און דערנאָך "Apply"

🔗 D-Link

  1. קלאָגן אַריין אין פּאַנעל (192.168.0.1)
  2. גיי צו: אַוואַנסירט → פּאָרט פאָרווערדינג
  3. דריק "Add"
  4. פֿילן:
    • Rule Name: כלל נאָמען
    • External Port: אויסערדיקער פּאָרט
    • Internal Port: אינערלעכער פּאָרט
    • Internal IP: מיטלס IP
    • Schedule: אייביק
    • Inbound Filter: Allow All
  5. היט סעטטינגס

📶 Zyxel Keenetic

  1. קלאָגן אַריין אין פּאַנעל (my.keenetic.net אָדער 192.168.1.1)
  2. גיי צו: זיכערהייט → פּאָרט פאָרווערדינג
  3. דריק "לייג צו כלל"
  4. פֿילן:
    • באַשרייבונג: דינסט נאָמען
    • צובינד: אינטערנעט
    • פּראָטאָקאָל: TCP/UDP
    • אויסערדיקער פּאָרט: פּאָרט פֿאַר אויסערדיקע צוטריט
    • ציל IP: מיטלס לאָקאַל IP
    • אינערלעכער פּאָרט: פּאָרט אויף מיטל
  5. היט ענדערונגען

✅ קאָנטראָלירן פּאָרט פאָרווערדינג אַרבעט

  1. ניצן אָנליין דינסטן

    וועבזייטלען צו קאָנטראָלירן עפענעפֿע פּאָרץ:

    • portchecker.co
    • canyouseeme.org

  2. קאָנטראָל דורך באַפֿעל שורה

    קאָנטראָלירן פֿון אן אנדער קאָמפּיוטער:

    # ניצן telnet אָדער nc (netcat)
telnet דיין-אויסערדיקער-IP פּאָרט
# אָדער ניצן nmap
nmap -p פּאָרט דיין-אויסערדיקער-IP

  3. דינסט טעסט

    פּרובירן צו פאַרבינדן צו דיין דינסט פֿון אן אנדער מיטל אַרויס פֿון דיין לאָקאַל נעץ (צ.ב.ש דורך מאבילער אינטערנעט).

✅ סימנים פֿון מצליח שטעלן

  • פּאָרט איז ווי "עפענעפֿע" אין אָנליין קאָנטראָלירערס
  • דו קענסט פאַרבינדן צו דינסט פֿון אויסן
  • דינסט אַרבעט סטאביל און אָן אונטערבריכונגען

🔧 לייזן פּראָבלעמען וואָס טרעפֿן זיך

🚫 פּאָרט נאָך פאַרשלאָסן

  • קאָנטראָלירן פֿייערוואָל: לייג אויסנאַם אין Windows פֿייערוואָל אָדער אנדערע פֿייערוואָל
  • רעבאָאָט ראַוטער: מאל סעטטינגס ווערן נאָר געווענדט נאָך רעבאָאָט
  • קאָנטראָלירן IP אַדרעס: זיכער זיין מיטל האט ריכטיק שטעטיק IP
  • דיזייבאַל UPnP: מאל UPnP קאָנפליקטן מיט מאַנואַל פּאָרט סעטטינגס

⚠️ פּראָווידערס דינאַמיש IP

אויב דו האסט דינאַמיש IP, עס קען זיך ענדערן. לייזונגען:

  • ניצן DDNS: דינסטן ווי No-IP, DuckDNS
  • מאָניטאָר IP: דערהייַנטיקן אינפֿאָרמאַציע וועגן היינטיקע IP ריגולירמעסיק
  • שטעטיק IP: בעטן שטעטיק IP פֿון פּראָווידער (באַצאָלט)

🔒 צוויי NAT

פּראָבלעם טרעפט זיך ווען מער ווי איין ראַוטער אין דיין נעץ:

  • ברידזש מאָדע: שטעל איינעם פון די ראַוטערס אין ברידזש מאָדע
  • DMZ: שטעל DMZ אויף הויפּט ראַוטער פֿאַר צווייטער ראַוטער
  • פאָרווערדינג אויף ביידע: שטעל פאָרווערדינג אויף ביידע ראַוטערס

🛡️ זיכערהייט שריט

🔐 וויכטיקע כּללים

  1. עפען נאָר נייטיקע פּאָרץ - יעדע עפענעפֿער פּאָרט פאַרגרעסערט אַטאַק פלאַך
  2. ניצן קאָמפּלעקס פּאַראָל אויף אַלע דינסטן וואָס קענען ווערן צוטריטן פֿון אויסן
  3. דערהייַנטיקן סאָפֿטווער ריגולירמעסיק - ווייַק זיך פֿון שוואַכקייטן
  4. מאָניטאָר קאַנעקשאַנז ניצן ראַוטער לאָגס
  5. באַטראַכטן VPN ווי אַ מער זיכערער אַלטערנאַטיוו צו פּאָרט פאָרווערדינג

🎯 בעסטע פיר

  • ניצן ניט-סטאַנדאַרט פּאָרץ צו רעדוצירן סקאַנערס אויפמערקזאַמקייט
  • שטעלן אַוטאָ דיזייבאַל פון פּאָרץ לויט פּלאַן
  • ריגולירמעסיק קאָנטראָל פון לאָגס פֿאַר פֿאַרדאַכטיק אַקטיוויטעט
  • ניצן fail2ban אָדער ענלעכע צו בלאקירן פֿאַרדאַכטיק IPs
