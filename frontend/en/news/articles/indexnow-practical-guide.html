IndexNow: Practical Implementation & Tips

September 15, 2025
Why this matters: deliver updates to search engines instantly, use changed‑only effectively, understand how IndexNow compares to sitemap ping, and avoid common mistakes.

1) What IndexNow is

IndexNow is a protocol to notify search engines about URL changes (create/update/delete). Supported by Bing, Yandex, Seznam and others. Benefits: lower latency to index and reduced crawl noise.

2) Key verification & basic ping

  1. Generate a key (GUID) and host YOUR_KEY.txt at the site root (/.well-known/ or domain root) containing the key string.
  2. Ping the endpoint with URLs and the key. Single URL example:
POST https://api.indexnow.org/indexnow Content-Type: application/json { "host": "www.serpmonn.ru", "key": "YOUR_KEY", "urlList": [ "https://www.serpmonn.ru/frontend/en/news/articles/utm-complete-guide.html" ] }

For batches, send up to 10,000 URLs per request or use a file feed with a URL of the changes list.

3) Changed‑only mode

Only ping URLs that actually changed since the last commit/deploy to reduce noise and quota usage.

  • Collect changed paths from VCS (git diff) or build artifacts.
  • Filter to public routes (exclude node_modules, .git, private dirs).
  • Group by domain/language to avoid irrelevant URLs.
Have a fallback: if changed‑only is empty (e.g., config hotfix), ping key sitemaps.

4) IndexNow vs sitemap ping

  • Speed: IndexNow pushes specific URLs immediately; sitemap ping only signals a sitemap change.
  • Noise: IndexNow is precise; sitemaps may trigger broader crawls.
  • Reliability: Use both — sitemaps for coverage, IndexNow for immediacy.

5) Common mistakes

  • Key not publicly accessible or mismatch with the key value.
  • Pinging private/404 pages, non‑canonical URLs with parameters.
  • Excessive repeats of the same URLs (throttle/dedupe).
  • Ignoring multilingual pages (ping each localized variant).

6) FAQ & checklist

How many URLs per batch?

Up to 10,000 — group by type/language.

Should deletions be pinged?

Yes — to speed up deindexing.

How to verify it works?

Check API responses (200/202), crawl spikes in server logs, and faster appearance of new URLs.

Rollout checklist

  • Key hosted and reachable via HTTPS.
  • Changed‑only path collector wired into CI/CD.
  • Deduping and throttling repeats.
  • Fallback to sitemap ping when batch is empty.
