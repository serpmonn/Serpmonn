IndexNow: Praktik Tətbiq və Tövsiyələr

15 sentyabr 2025 • 5 dəqiqə oxu
Niyə oxumaq: axtarış sistemlərinə yeniləmələri necə tez çatdırmaq, nə vaxt yalnız dəyişdirilmiş istifadə etmək, IndexNow sitemap ping-dən necə fərqlənir və ümumi səhvlərdən necə qaçmaq.

1) IndexNow nədir

IndexNow, axtarış sistemlərinə URL dəyişiklikləri (yaradılma, yeniləmə, silinmə) barədə dərhal məlumat verməyə imkan verən protokoldur. Bing, Yandex, Seznam və digər xidmətlər tərəfindən dəstəklənir. Üstünlüyü minimal indeksləşdirmə gecikməsi və daha az crawl səs-küyüdür.

2) Açar quraşdırma və əsas ping

  1. Bir açar (GUID) yaradın və YOUR_KEY.txt faylını saytın kökündə (/.well-known/ və ya domen kökü) açar sətri ilə yerləşdirin.
  2. URL və açar ilə endpoint-ə ping göndərin. Nümunə (tək URL):
POST https://api.indexnow.org/indexnow Content-Type: application/json { "host": "www.serpmonn.ru", "key": "YOUR_KEY", "urlList": [ "https://www.serpmonn.ru/frontend/knowledge-base/articles/utm-complete-guide.html" ] }
Partiyalar üçün bir sorğuda 10.000-ə qədər URL göstərin və ya dəyişiklik jurnalı faylının URL ilə fayl axınından istifadə edin.

3) Yalnız dəyişdirilmiş rejim

Fikir son commit/deployment-da faktiki olaraq dəyişmiş fayllara yalnız ping göndərməkdir. Bu, səs-küyü və kvota istifadəsini azaldır.

  • VCS-dən (git diff) və ya build artefaktlarından dəyişdirilmiş yolların siyahısını əldə edin.
  • İctimai marşrutlara görə filtrləyin (node_modules, .git, şəxsi qovluqlar istisna olmaqla).
  • Əlaqəsiz URL-lər göndərməmək üçün domen/dillər üzrə qruplaşdırın.
İstehsalda ehtiyat olmaq faydalıdır: yalnız dəyişdirilmiş boşdursa (məsələn, konfiqurasiyalar üçün hotfix), vacib sayt xəritələrinə ping göndərin.

4) IndexNow vs sitemap ping

  • Sürət: IndexNow yeniləmələri daha sürətli çatdırır, crawl dövrünü gözləmir; sitemap ping sayt xəritəsinin olmasını bildirir, lakin konkret URL-də dəyişikliyi yox.
  • Səs-küy: IndexNow - dəyişdirilmiş URL-lərə hədəflənmiş; sitemap - bütün xəritənin skanı (crawl büdcəsi üçün daha bahalı).
  • Etibarlılıq: Sayt xəritələri əsas olaraq qalır; IndexNow tamamlayır və sürətləndirir. Hər ikisini istifadə etmək daha yaxşıdır.

5) Ümumi səhvlər

  • Açar ictimai URL vasitəsilə əlçatan deyil və ya key parametri ilə uyğun gəlmir.
  • Şəxsi/404 səhifələrə ping, kanonikallaşdırma olmadan parametrlər.
  • Eyni URL-lərin çox tez-tez təkrarlanması (təkrarları məhdudlaşdırın).
  • Çoxdilliliyi görməməzlikdən gəlmək (müvafiq lokalizə edilmiş versiyaya ping göndərmək lazımdır).

6) FAQ və kontrol siyahısı

Bir anda neçə URL göndərmək lazımdır?

Bir partiyada 10.000-ə qədər - optimal olaraq növ/dil üzrə qruplaşdırın.

Silinmələrə ping göndərməliyəm?

Bəli - silinmiş səhifələrin URL-lərini deindexləşdirməni sürətləndirmək üçün ötürün.

Necə işlədiyini təsdiqləmək olar?

API cavab jurnalları (200/202), server jurnallarında crawl pikleri, yeni URL-lərin daha tez görünməsi.

  • Açar yerləşdirilib və HTTPS vasitəsilə əlçatandır.
  • CI/CD-dən yalnız dəyişdirilmiş yolları toplayın.
  • Təkrarların təkrarının aradan qaldırılması və məhdudlaşdırılması.
  • Boş partiyada sitemap ping-ə ehtiyat.
: IndexNow yeni və dəyişdirilmiş səhifələrin indeksləşdirilməsini əhəmiyyətli dərəcədə sürətləndirir. Ənənəvi sayt xəritələri ilə birlikdə SEO optimallaşdırılması üçün güclü vasitədir.
