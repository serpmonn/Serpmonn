IndexNow: Implementação prática e dicas

15 de setembro de 2025 • 5 minutos de leitura
Porquê ler: como entregar atualizações rapidamente aos motores de busca, quando usar apenas-alteradas, como o IndexNow difere do sitemap ping e como evitar erros comuns.

1) O que é o IndexNow

IndexNow é um protocolo que permite notificar instantaneamente os motores de busca sobre alterações de URL (criação, atualização, exclusão). Suportado pelo Bing, Yandex, Seznam e outros serviços. A vantagem é o atraso mínimo de indexação e menos ruído de rastreamento.

2) Configuração da chave e ping básico

  1. Gere uma chave (GUID) e coloque o ficheiro YOUR_KEY.txt na raiz do site (/.well-known/ ou raiz do domínio) com a string da chave.
  2. Envie ping para o endpoint com o URL e a chave. Exemplo (URL único):
POST https://api.indexnow.org/indexnow Content-Type: application/json { "host": "www.serpmonn.ru", "key": "YOUR_KEY", "urlList": [ "https://www.serpmonn.ru/frontend/news/articles/utm-complete-guide.html" ] }
Para lotes, especifique até 10.000 URLs num pedido ou use um feed de ficheiro com um URL de ficheiro de registo de alterações.

3) Modo apenas-alteradas

A ideia é enviar ping apenas para ficheiros realmente alterados no último commit/implementação. Isto reduz o ruído e o uso de quota.

  • Obtenha a lista de caminhos alterados do VCS (git diff) ou artefactos de construção.
  • Filtre por rotas públicas (excluindo node_modules, .git, pastas privadas).
  • Agrupe por domínio/idioma para evitar enviar URLs irrelevantes.
Em produção, é útil ter um fallback: se apenas-alteradas estiver vazio (ex.: correção de configurações), envie ping para sitemaps importantes.

4) IndexNow vs sitemap ping

  • Velocidade: IndexNow entrega atualizações mais rápido, não espera pelo ciclo de rastreamento; sitemap ping sinaliza a presença de um sitemap, mas não alterações em URL específico.
  • Ruído: IndexNow - direcionado a URLs alteradas; sitemap - varrimento de todo o mapa (mais caro para o orçamento de rastreamento).
  • Fiabilidade: Sitemaps permanecem a base; IndexNow complementa e acelera. É melhor usar ambos.

5) Erros comuns

  • A chave não está acessível via URL público ou não corresponde ao parâmetro key.
  • Enviar ping para páginas privadas/404, parâmetros sem canonicalização.
  • Repetições muito frequentes dos mesmos URLs (limite as repetições).
  • Ignorar multilingue (precisa enviar ping para a versão localizada correspondente).

6) FAQ e lista de verificação

Quantos URLs enviar de uma vez?

Até 10.000 num lote - idealmente agrupados por tipo/idioma.

Devo enviar ping para exclusões?

Sim - passe os URLs das páginas excluídas para acelerar a desindexação.

Como confirmar que está a funcionar?

Registos de resposta da API (200/202), picos de rastreamento nos registos do servidor, aparecimento mais rápido de novos URLs.

  • Chave colocada e acessível via HTTPS.
  • Recolher caminhos apenas-alteradas do CI/CD.
  • Deduplicação e limitação de repetições.
  • Fallback para sitemap ping em lote vazio.
: IndexNow acelera significativamente a indexação de páginas novas e alteradas. Combinado com sitemaps tradicionais, é uma ferramenta poderosa para otimização de SEO.
